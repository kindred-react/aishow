### Q1: 单例模式有几种实现方式？各自的优缺点是什么？ ⭐

**答案：**

**1. 饿汉式（线程安全）**
```java
public class Singleton {
    private static final Singleton INSTANCE = new Singleton();

    private Singleton() {}

    public static Singleton getInstance() {
        return INSTANCE;
    }
}
```
- 优点：线程安全、实现简单
- 缺点：类加载时就创建实例，可能浪费资源

**2. 懒汉式（线程不安全）**
```java
public class Singleton {
    private static Singleton instance;

    private Singleton() {}

    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}
```
- 优点：延迟加载
- 缺点：线程不安全，多线程下可能创建多个实例

**3. 懒汉式 + synchronized（线程安全，性能差）**
```java
public static synchronized Singleton getInstance() {
    if (instance == null) {
        instance = new Singleton();
    }
    return instance;
}
```
- 优点：线程安全
- 缺点：每次获取实例都要加锁，性能差

**4. 双重检查锁（DCL，推荐）**
```java
public class Singleton {
    // volatile防止指令重排序
    private static volatile Singleton instance;

    private Singleton() {}

    public static Singleton getInstance() {
        if (instance == null) {                    // 第一次检查（无锁）
            synchronized (Singleton.class) {
                if (instance == null) {            // 第二次检查（有锁）
                    instance = new Singleton();     // 可能发生指令重排序
                }
            }
        }
        return instance;
    }
}
```
- 为什么需要volatile？`new Singleton()`不是原子操作：
  1. 分配内存空间
  2. 初始化对象
  3. 将引用指向内存空间
  步骤2和3可能被重排序，导致其他线程拿到未初始化的对象

**5. 静态内部类（推荐）**
```java
public class Singleton {
    private Singleton() {}

    private static class Holder {
        private static final Singleton INSTANCE = new Singleton();
    }

    public static Singleton getInstance() {
        return Holder.INSTANCE;
    }
}
```
- 利用类加载机制保证线程安全
- 延迟加载（Holder类只在getInstance时才加载）
- 不需要加锁，性能好

**6. 枚举（最佳实践）**
```java
public enum Singleton {
    INSTANCE;

    public void doSomething() {
        System.out.println("Singleton method");
    }
}
```
- 优点：线程安全、防止反射攻击、防止序列化/反序列化创建新对象
- 缺点：不支持延迟加载、不够灵活

| 实现方式 | 线程安全 | 延迟加载 | 防止反射 | 防止序列化 |
|---------|---------|---------|---------|-----------|
| 饿汉式 | 是 | 否 | 否 | 否 |
| 懒汉式 | 否 | 是 | 否 | 否 |
| DCL | 是 | 是 | 否 | 否 |
| 静态内部类 | 是 | 是 | 否 | 否 |
| 枚举 | 是 | 否 | 是 | 是 |

### Q2: 工厂模式有几种？分别如何实现？ ⭐⭐

**答案：**

**1. 简单工厂（静态工厂方法）**

不属于GoF 23种设计模式，但很常用。

```java
public class ShapeFactory {

    public static Shape createShape(String type) {
        switch (type) {
            case "circle": return new Circle();
            case "rectangle": return new Rectangle();
            case "triangle": return new Triangle();
            default: throw new IllegalArgumentException("Unknown shape: " + type);
        }
    }
}

// 使用
Shape circle = ShapeFactory.createShape("circle");
circle.draw();
```
- 优点：简单易用
- 缺点：违反开闭原则，新增类型需要修改工厂类

**2. 工厂方法模式**

每个产品对应一个工厂。

```java
// 抽象工厂
public interface ShapeFactory {
    Shape createShape();
}

// 具体工厂
public class CircleFactory implements ShapeFactory {
    @Override
    public Shape createShape() {
        return new Circle();
    }
}

public class RectangleFactory implements ShapeFactory {
    @Override
    public Shape createShape() {
        return new Rectangle();
    }
}

// 使用
ShapeFactory factory = new CircleFactory();
Shape shape = factory.createShape();
shape.draw();
```
- 优点：符合开闭原则，新增产品只需新增工厂类
- 缺点：类数量增加

**3. 抽象工厂模式**

创建一族相关产品。

```java
// 抽象工厂接口
public interface UIFactory {
    Button createButton();
    TextField createTextField();
    Checkbox createCheckbox();
}

// 具体工厂 - Windows风格
public class WindowsUIFactory implements UIFactory {
    @Override
    public Button createButton() { return new WindowsButton(); }

    @Override
    public TextField createTextField() { return new WindowsTextField(); }

    @Override
    public Checkbox createCheckbox() { return new WindowsCheckbox(); }
}

// 具体工厂 - Mac风格
public class MacUIFactory implements UIFactory {
    @Override
    public Button createButton() { return new MacButton(); }

    @Override
    public TextField createTextField() { return new MacTextField(); }

    @Override
    public Checkbox createCheckbox() { return new MacCheckbox(); }
}

// 使用
UIFactory factory = new WindowsUIFactory();
Button button = factory.createButton();
TextField textField = factory.createTextField();
```
- 优点：可以创建一族产品，保证产品之间的兼容性
- 缺点：新增产品类型需要修改所有工厂接口

**三者对比：**

| 模式 | 工厂数量 | 产品等级 | 产品族 |
|------|---------|---------|-------|
| 简单工厂 | 1个 | 多个 | 1个 |
| 工厂方法 | 多个 | 1个 | 1个 |
| 抽象工厂 | 多个 | 多个 | 多个 |

### Q3: 建造者模式如何实现？Lombok的@Builder原理是什么？ ⭐⭐

**答案：**

**建造者模式**用于将复杂对象的构建与表示分离，使得同样的构建过程可以创建不同的表示。

**传统实现：**
```java
public class HttpRequest {
    private String url;
    private String method;
    private Map<String, String> headers;
    private String body;
    private int timeout;

    private HttpRequest(Builder builder) {
        this.url = builder.url;
        this.method = builder.method;
        this.headers = builder.headers;
        this.body = builder.body;
        this.timeout = builder.timeout;
    }

    public static class Builder {
        private String url;
        private String method = "GET";
        private Map<String, String> headers = new HashMap<>();
        private String body;
        private int timeout = 30000;

        public Builder url(String url) {
            this.url = url;
            return this;
        }

        public Builder method(String method) {
            this.method = method;
            return this;
        }

        public Builder addHeader(String key, String value) {
            this.headers.put(key, value);
            return this;
        }

        public Builder body(String body) {
            this.body = body;
            return this;
        }

        public Builder timeout(int timeout) {
            this.timeout = timeout;
            return this;
        }

        public HttpRequest build() {
            return new HttpRequest(this);
        }
    }
}

// 使用（链式调用）
HttpRequest request = new HttpRequest.Builder()
    .url("https://api.example.com/users")
    .method("POST")
    .addHeader("Content-Type", "application/json")
    .body("{\"name\":\"张三\"}")
    .timeout(5000)
    .build();
```

**Lombok @Builder原理：**
```java
@Builder
public class User {
    private Long id;
    private String name;
    private Integer age;
}
```
Lombok在编译时自动生成：
- 全参私有构造方法
- 静态内部类`UserBuilder`，包含所有字段的setter方法（返回Builder自身）
- `build()`方法调用构造方法创建对象
- `builder()`静态方法创建Builder实例

**注意事项：**
- `@Builder.Default`可以设置默认值
- `@Builder(toBuilder = true)`可以基于已有对象创建Builder
- 与JPA实体类配合时需要加`@NoArgsConstructor`和`@AllArgsConstructor`

### Q4: 原型模式是什么？深拷贝和浅拷贝有什么区别？ ⭐⭐

**答案：**

**原型模式**通过拷贝已有对象来创建新对象，而不是通过new。

**浅拷贝 vs 深拷贝：**

| 维度 | 浅拷贝 | 深拷贝 |
|------|--------|--------|
| 基本类型 | 复制值 | 复制值 |
| 引用类型 | 复制引用（共享对象） | 创建新对象 |
| 实现方式 | `Object.clone()` | 序列化/递归拷贝 |
| 影响范围 | 修改引用类型会影响原对象 | 修改不影响原对象 |

**浅拷贝实现：**
```java
public class Prototype implements Cloneable {
    private String name;
    private List<String> hobbies;

    @Override
    protected Prototype clone() throws CloneNotSupportedException {
        return (Prototype) super.clone();
    }
}

// 使用
Prototype original = new Prototype();
original.setHobbies(List.of("reading", "coding"));
Prototype copy = original.clone();
copy.getHobbies().add("swimming");  // 影响原对象！
```

**深拷贝实现（序列化方式）：**
```java
public class DeepPrototype implements Serializable {
    private String name;
    private List<String> hobbies;

    public DeepPrototype deepClone() {
        try {
            ByteArrayOutputStream bos = new ByteArrayOutputStream();
            ObjectOutputStream oos = new ObjectOutputStream(bos);
            oos.writeObject(this);

            ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
            ObjectInputStream ois = new ObjectInputStream(bis);
            return (DeepPrototype) ois.readObject();
        } catch (Exception e) {
            throw new RuntimeException("Deep clone failed", e);
        }
    }
}
```

**深拷贝实现（JSON方式）：**
```java
public <T> T deepClone(T object, Class<T> clazz) {
    return JSON.parseObject(JSON.toJSONString(object), clazz);
}
// 注意：JSON 方式不支持循环引用和 transient 字段，且需要 fastjson/jackson 依赖
```

**适用场景：**
- 创建对象成本较大（需要大量数据库查询或IO操作）
- 需要创建大量相似对象
- 保护性拷贝（如`ArrayList.clone()`）

### Q5: 代理模式有几种？动态代理的原理是什么？ ⭐⭐

**答案：**

**1. 静态代理**
```java
public interface Subject {
    void doSomething();
}

public class RealSubject implements Subject {
    @Override
    public void doSomething() {
        System.out.println("RealSubject doing something");
    }
}

public class StaticProxy implements Subject {
    private final Subject target;

    public StaticProxy(Subject target) {
        this.target = target;
    }

    @Override
    public void doSomething() {
        System.out.println("Before");
        target.doSomething();
        System.out.println("After");
    }
}
```
- 缺点：每个接口都需要手动编写代理类

**2. JDK动态代理（基于接口）**
```java
public class JdkProxy implements InvocationHandler {

    private final Object target;

    public JdkProxy(Object target) {
        this.target = target;
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("Before: " + method.getName());
        Object result = method.invoke(target, args);
        System.out.println("After: " + method.getName());
        return result;
    }

    @SuppressWarnings("unchecked")
    public <T> T getProxy() {
        return (T) Proxy.newProxyInstance(
            target.getClass().getClassLoader(),
            target.getClass().getInterfaces(),
            this
        );
    }
}

// 使用
Subject proxy = new JdkProxy(new RealSubject()).getProxy();
proxy.doSomething();
```

**JDK动态代理原理：**
1. `Proxy.newProxyInstance()`在运行时生成代理类的字节码
2. 代理类继承`Proxy`，实现目标接口
3. 代理类中每个方法都调用`InvocationHandler.invoke()`
4. `invoke()`中通过反射调用目标方法

**3. CGLIB动态代理（基于继承）**
```java
public class CglibProxy implements MethodInterceptor {

    @Override
    public Object intercept(Object obj, Method method, Object[] args,
                            MethodProxy methodProxy) throws Throwable {
        System.out.println("Before: " + method.getName());
        Object result = methodProxy.invokeSuper(obj, args);
        System.out.println("After: " + method.getName());
        return result;
    }

    @SuppressWarnings("unchecked")
    public <T> T getProxy(Class<T> clazz) {
        Enhancer enhancer = new Enhancer();
        enhancer.setSuperclass(clazz);
        enhancer.setCallback(this);
        return (T) enhancer.create();
    }
}

// 使用
RealSubject proxy = new CglibProxy().getProxy(RealSubject.class);
proxy.doSomething();
```

**JDK代理 vs CGLIB代理：**

| 维度 | JDK动态代理 | CGLIB动态代理 |
|------|-----------|--------------|
| 原理 | 基于接口（实现接口） | 基于继承（生成子类） |
| 要求 | 目标类必须实现接口 | 目标类不能是final |
| 性能 | JDK 8+性能已大幅提升 | 创建代理稍慢，执行稍快 |
| Spring默认 | Spring Boot 2.x（Spring 5+）默认使用 CGLIB 代理（spring.aop.proxy-target-class=true），无论目标是否实现接口 | 目标无接口时使用 |

### Q6: 策略模式如何消除if-else？ ⭐⭐

**答案：**

**策略模式**定义一系列算法，将每个算法封装起来，使它们可以互相替换。

**传统if-else写法：**
```java
public double calculatePrice(String type, double price) {
    if ("normal".equals(type)) {
        return price;
    } else if ("vip".equals(type)) {
        return price * 0.8;
    } else if ("svip".equals(type)) {
        return price * 0.7;
    } else if ("coupon".equals(type)) {
        return price - 50;
    }
    throw new IllegalArgumentException("Unknown type");
}
```

**策略模式重构：**
```java
// 策略接口
public interface PricingStrategy {
    double calculate(double price);
    String getType();
}

// 具体策略
@Component
public class NormalPricingStrategy implements PricingStrategy {
    @Override
    public double calculate(double price) { return price; }
    @Override
    public String getType() { return "normal"; }
}

@Component
public class VipPricingStrategy implements PricingStrategy {
    @Override
    public double calculate(double price) { return price * 0.8; }
    @Override
    public String getType() { return "vip"; }
}

@Component
public class SvipPricingStrategy implements PricingStrategy {
    @Override
    public double calculate(double price) { return price * 0.7; }
    @Override
    public String getType() { return "svip"; }
}

// 策略工厂（利用Spring自动注入）
@Component
public class PricingStrategyFactory {

    private final Map<String, PricingStrategy> strategyMap;

    @Autowired
    public PricingStrategyFactory(List<PricingStrategy> strategies) {
        this.strategyMap = strategies.stream()
            .collect(Collectors.toMap(PricingStrategy::getType, Function.identity()));
    }

    public PricingStrategy getStrategy(String type) {
        PricingStrategy strategy = strategyMap.get(type);
        if (strategy == null) {
            throw new IllegalArgumentException("Unknown type: " + type);
        }
        return strategy;
    }
}

// 使用
@Service
public class PricingService {
    @Autowired
    private PricingStrategyFactory strategyFactory;

    public double calculatePrice(String type, double price) {
        return strategyFactory.getStrategy(type).calculate(price);
    }
}
```

**Spring中的策略模式应用：**
- `Resource`接口的不同实现（`ClassPathResource`、`FileSystemResource`等）
- `TaskExecutor`的不同实现（`SyncTaskExecutor`、`ThreadPoolTaskExecutor`等）
- `ViewResolver`的不同实现

### Q7: 观察者模式是什么？Spring事件机制如何使用？ ⭐⭐

**答案：**

**观察者模式**定义对象间一对多的依赖关系，当一个对象状态改变时，所有依赖它的对象都会收到通知。

**传统实现：**
```java
// 观察者接口
public interface EventListener {
    void onEvent(Event event);
}

// 被观察者
public class EventBus {
    private final List<EventListener> listeners = new ArrayList<>();

    public void register(EventListener listener) {
        listeners.add(listener);
    }

    public void unregister(EventListener listener) {
        listeners.remove(listener);
    }

    public void publish(Event event) {
        listeners.forEach(listener -> listener.onEvent(event));
    }
}
```

**Spring事件机制：**

**1. 定义事件**
```java
public class OrderCreatedEvent extends ApplicationEvent {
    private final Order order;

    public OrderCreatedEvent(Object source, Order order) {
        super(source);
        this.order = order;
    }

    public Order getOrder() { return order; }
}
```

**2. 发布事件**
```java
@Service
public class OrderService {

    @Autowired
    private ApplicationEventPublisher eventPublisher;

    public void createOrder(Order order) {
        // 保存订单
        orderRepository.save(order);
        // 发布事件
        eventPublisher.publishEvent(new OrderCreatedEvent(this, order));
    }
}
```

**3. 监听事件**
```java
@Component
public class OrderEventListener {

    // 注解方式
    @EventListener
    public void handleOrderCreated(OrderCreatedEvent event) {
        System.out.println("Order created: " + event.getOrder().getId());
        // 发送通知、更新库存、记录日志等
    }

    // 异步监听
    @Async
    @EventListener
    public void sendNotification(OrderCreatedEvent event) {
        emailService.sendOrderConfirmation(event.getOrder());
    }

    // 条件监听
    @EventListener(condition = "#event.order.amount > 1000")
    public void handleLargeOrder(OrderCreatedEvent event) {
        System.out.println("Large order alert: " + event.getOrder().getAmount());
    }
}
```

**4. 事务绑定事件**
```java
// 事务提交后才执行
@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
public void afterCommit(OrderCreatedEvent event) {
    // 事务成功提交后执行
}
```

**Spring事件机制优势：**
- 松耦合：发布者不需要知道监听者的存在
- 异步支持：`@Async` + `@EventListener`
- 事务支持：`@TransactionalEventListener`
- 过滤支持：`condition`表达式

### Q8: 模板方法模式是什么？有哪些应用？ ⭐⭐

**答案：**

**模板方法模式**在父类中定义算法骨架，将某些步骤延迟到子类实现。

```java
public abstract class AbstractDataParser {

    // 模板方法（定义算法骨架，final防止子类修改）
    public final void parse() {
        openFile();
        readData();
        processData();
        closeFile();
    }

    // 具体方法（所有子类共用）
    private void openFile() {
        System.out.println("Opening file...");
    }

    private void closeFile() {
        System.out.println("Closing file...");
    }

    // 抽象方法（子类实现）
    protected abstract void readData();

    // 钩子方法（子类可选实现）
    protected void processData() {
        // 默认实现（空操作）
    }
}

public class JsonParser extends AbstractDataParser {
    @Override
    protected void readData() {
        System.out.println("Reading JSON data...");
    }

    @Override
    protected void processData() {
        System.out.println("Processing JSON data...");
    }
}

public class XmlParser extends AbstractDataParser {
    @Override
    protected void readData() {
        System.out.println("Reading XML data...");
    }
}
```

**Spring中的应用：**

**1. JdbcTemplate**
```java
jdbcTemplate.query("SELECT * FROM user", new RowMapper<User>() {
    @Override
    public User mapRow(ResultSet rs, int rowNum) throws SQLException {
        User user = new User();
        user.setId(rs.getLong("id"));
        user.setName(rs.getString("name"));
        return user;
    }
});
// JdbcTemplate定义了获取连接、创建Statement、执行查询、
// 关闭连接等模板流程，用户只需实现结果映射
```

**2. RedisTemplate**
```java
redisTemplate.execute(new SessionCallback<Object>() {
    @Override
    public Object execute(RedisOperations operations) throws DataAccessException {
        operations.multi();
        operations.opsForValue().set("key1", "value1");
        operations.opsForValue().set("key2", "value2");
        return operations.exec();
    }
});
```

**3. HttpServlet**
```java
public class MyServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) {
        // 处理GET请求
    }

    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) {
        // 处理POST请求
    }
}
// HttpServlet.service()是模板方法，根据请求方法分发到doGet/doPost
```

### Q9: 责任链模式是什么？有哪些应用场景？ ⭐⭐⭐

**答案：**

**责任链模式**将请求的发送者和接收者解耦，使多个对象都有机会处理请求。将这些对象连成一条链，沿链传递请求，直到有一个对象处理为止。

**通用实现：**
```java
public abstract class Handler {

    private Handler next;

    public Handler setNext(Handler next) {
        this.next = next;
        return next;  // 支持链式调用
    }

    public abstract boolean handle(Request request);

    protected boolean handleNext(Request request) {
        return next != null && next.handle(request);
    }
}

public class AuthHandler extends Handler {
    @Override
    public boolean handle(Request request) {
        if (!request.isAuthenticated()) {
            System.out.println("Authentication failed");
            return false;
        }
        System.out.println("Authentication passed");
        return handleNext(request);
    }
}

public class RateLimitHandler extends Handler {
    @Override
    public boolean handle(Request request) {
        if (request.getRate() > 100) {
            System.out.println("Rate limit exceeded");
            return false;
        }
        System.out.println("Rate limit passed");
        return handleNext(request);
    }
}

public class LogHandler extends Handler {
    @Override
    public boolean handle(Request request) {
        System.out.println("Logging request: " + request);
        return handleNext(request);
    }
}

// 使用
Handler chain = new AuthHandler()
    .setNext(new RateLimitHandler())
    .setNext(new LogHandler());
chain.handle(request);
```

**典型应用场景：**

**1. Servlet Filter**
```java
@WebFilter(urlPatterns = "/*")
public class AuthFilter implements Filter {
    @Override
```

### Q10: 装饰器模式是什么？Java IO中有哪些应用？ ⭐⭐

**答案：**

**装饰器模式**动态地给对象添加额外职责，比继承更灵活。

**核心结构：**
- Component：抽象构件
- ConcreteComponent：具体构件
- Decorator：抽象装饰器（持有一个Component引用）
- ConcreteDecorator：具体装饰器

```java
// 抽象构件
public interface Coffee {
    String getDescription();
    double getCost();
}

// 具体构件

    @Override
    public String getDescription() { return "Espresso"; }

    @Override
    public double getCost() { return 25.0; }
}

public abstract class CoffeeDecorator implements Coffee {
    protected final Coffee coffee;

    public CoffeeDecorator(Coffee coffee) {
        this.coffee = coffee;
    }
}

// 具体装饰器
public class MilkDecorator extends CoffeeDecorator {
    public MilkDecorator(Coffee coffee) { super(coffee); }

    @Override
    public String getDescription() { return coffee.getDescription() + ", Milk"; }

    @Override
    public double getCost() { return coffee.getCost() + 5.0; }
}

public class MochaDecorator extends CoffeeDecorator {
    public MochaDecorator(Coffee coffee) { super(coffee); }

    @Override
    public String getDescription() { return coffee.getDescription() + ", Mocha"; }

    @Override
    public double getCost() { return coffee.getCost() + 8.0; }
}
```

### Q12: 享元模式是什么？Java中有哪些应用？ ⭐⭐

**答案：**

**享元模式**运用共享技术有效地支持大量细粒度的对象，减少内存使用。

**核心结构：**
- Flyweight（抽象享元）：定义接口
- ConcreteFlyweight（具体享元）：实现接口，存储内部状态
- FlyweightFactory（享元工厂）：管理享元池

```java
public class Tree {
    private final String type;    // 内部状态（共享）
    private final String color;   // 内部状态（共享）

    public Tree(String type, String color) {
        this.type = type;
        this.color = color;
    }

    public void display(int x, int y) {  // x, y是外部状态
        System.out.println(type + " tree at (" + x + "," + y + ") color=" + color);
    }
}

public class TreeFactory {
    private static final Map<String, Tree> TREE_POOL = new HashMap<>();

    public static Tree getTree(String type, String color) {
        String key = type + ":" + color;
        return TREE_POOL.computeIfAbsent(key, k -> new Tree(type, color));
    }

    public static int getPoolSize() {
        return TREE_POOL.size();
    }
}

// 使用
Tree t1 = TreeFactory.getTree("Oak", "Green");
Tree t2 = TreeFactory.getTree("Oak", "Green");
Tree t3 = TreeFactory.getTree("Pine", "Green");
System.out.println(t1 == t2);  // true（同一对象）
System.out.println(TreeFactory.getPoolSize());  // 2
```

### Q13: 组合模式是什么？如何实现树形结构？ ⭐⭐

**答案：**

**组合模式**将对象组合成树形结构，使单个对象和组合对象的使用具有一致性。

```java
// 抽象组件
public abstract class MenuComponent {
    protected String name;
    protected String url;

    public MenuComponent(String name, String url) {
        this.name = name;
        this.url = url;
    }

    public abstract void print();

    public abstract void add(MenuComponent component);
    public abstract void remove(MenuComponent component);
    public abstract MenuComponent getChild(int i);
}

// 叶子节点（菜单项）

    public MenuItem(String name, String url) {
        super(name, url);
    }

    @Override
    public void print() {
        System.out.println("  " + name + " : " + url);
    }

    @Override
    public void add(MenuComponent component) {
        throw new UnsupportedOperationException();
    }

    @Override
    public void remove(MenuComponent component) {
        throw new UnsupportedOperationException();
    }

    @Override
    public MenuComponent getChild(int i) {
        throw new UnsupportedOperationException();
    }
}

// 组合节点（子菜单）
public class Menu extends MenuComponent {
    private final List<MenuComponent> children = new ArrayList<>();

    public Menu(String name, String url) {
        super(name, url);
    }

    @Override
    public void print() {
        System.out.println(name + " : " + url);

    @Override
    public void add(MenuComponent component) {
        children.add(component);
    }

    @Override
    public void remove(MenuComponent component) {
        children.remove(component);
    }

    @Override
    public MenuComponent getChild(int i) {
        return children.get(i);

}
```

### Q14: 迭代器模式是什么？Java中如何使用？ ⭐

**答案：**

**迭代器模式**提供一种方法顺序访问聚合对象中的各个元素，而不暴露其内部表示。

```java
// 迭代器接口
public interface Iterator<T> {
    boolean hasNext();
    T next();
}

// 聚合接口
public interface Aggregate<T> {
    Iterator<T> iterator();
}

// 具体聚合
public class MyList<T> implements Aggregate<T> {
    private final List<T> items = new ArrayList<>();

    public void add(T item) { items.add(item); }

    @Override
    public Iterator<T> iterator() {
        return new MyIterator();
    }

    private class MyIterator implements Iterator<T> {
        private int index = 0;

        @Override
        public boolean hasNext() {
            return index < items.size();
        }

        @Override
        public T next() {
            return items.get(index++);
        }
    }
}
```

**Java中的迭代器：**
```java
// Collection接口继承了Iterable
// 所有集合类都可以使用for-each语法
List<String> list = Arrays.asList("A", "B", "C");

// 方式1：Iterator
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String item = it.next();
    if ("B".equals(item)) {
        it.remove();  // 迭代过程中删除元素
    }
}

// 方式2：for-each（语法糖）
for (String item : list) {
    System.out.println(item);
}

// 方式3：ListIterator（支持双向遍历）
ListIterator<String> listIt = list.listIterator();
while (listIt.hasNext()) {
    listIt.next();
}
while (listIt.hasPrevious()) {
    listIt.previous();
}
```

**注意事项：**
- 迭代器遍历时不能直接修改集合（ConcurrentModificationException）
- 需要通过迭代器的`remove()`方法删除元素
- 可以使用`CopyOnWriteArrayList`避免并发修改异常

```java
    public void ship(OrderContext context) {

    @Override
    public void confirm(OrderContext context) {

    }

    @Override
    public void cancel(OrderContext context) {
        context.setState(new CancelledState());
    }
}

    @Override
    public void pay(OrderContext context) {
        System.out.println("Order already paid");
    }

    @Override
    public void ship(OrderContext context) {
        context.setState(new ShippedState());
    }

    @Override
    public void confirm(OrderContext context) {
        System.out.println("Cannot confirm unshipped order");
    }

    @Override
    public void cancel(OrderContext context) {
        context.setState(new RefundingState());
    }
}

// 已发货状态
public class ShippedState implements OrderState {
    @Override
    public void pay(OrderContext context) {
        System.out.println("Order already paid");
    }

    @Override
    public void ship(OrderContext context) {
        System.out.println("Order already shipped");
    }

    @Override
    public void confirm(OrderContext context) {
        context.setState(new CompletedState());
    }

    @Override
    public void cancel(OrderContext context) {
        context.setState(new RefundingState());
    }
}

// 已完成状态

    @Override
```

### Q16: 命令模式是什么？有哪些应用场景？ ⭐⭐

**答案：**

**命令模式**将请求封装为对象，使你可以用不同的请求对客户进行参数化、排队或记录请求日志，以及支持撤销操作。

```java
// 命令接口
public interface Command {
    void execute();
    void undo();
}

// 接收者
public class TextEditor {
    private StringBuilder content = new StringBuilder();

    public void append(String text) {
        content.append(text);
    }

    public void delete(int length) {
        content.setLength(content.length() - length);
    }

    public String getContent() {
        return content.toString();
    }
}

// 具体命令
public class AppendCommand implements Command {
    private final TextEditor editor;
    private final String text;

    public AppendCommand(TextEditor editor, String text) {
        this.editor = editor;
        this.text = text;
    }

    @Override
    public void execute() {
        editor.append(text);
    }

    @Override
    public void undo() {
        editor.delete(text.length());
    }
}

// 调用者
public class CommandInvoker {
    private final Deque<Command> history = new ArrayDeque<>();

    public void executeCommand(Command command) {
        command.execute();
        history.push(command);
    }

    public void undo() {
        if (!history.isEmpty()) {
            Command command = history.pop();
            command.undo();
        }
    }
}

// 使用
TextEditor editor = new TextEditor();
CommandInvoker invoker = new CommandInvoker();

invoker.executeCommand(new AppendCommand(editor, "Hello "));
invoker.executeCommand(new AppendCommand(editor, "World "));
System.out.println(editor.getContent());  // "Hello World "

invoker.undo();
System.out.println(editor.getContent());  // "Hello "
```

**应用场景：**
- **GUI操作**：按钮点击、菜单操作
- **撤销/重做**：文本编辑器、IDE
- **宏命令**：批量操作
- **MQ消息**：将消息封装为命令对象
- **任务队列**：异步任务调度

### Q17: 中介者模式是什么？有哪些应用场景？ ⭐⭐

**答案：**

**中介者模式**用一个中介对象来封装一系列的对象交互，使各对象不需要显式地相互引用。

```java
// 中介者接口
public interface ChatMediator {
    void sendMessage(User user, String message);
    void addUser(User user);
}

// 具体中介者
public class ChatRoom implements ChatMediator {
    private final List<User> users = new ArrayList<>();

    @Override
    public void addUser(User user) {
        users.add(user);
    }

    @Override
    public void sendMessage(User sender, String message) {
        users.stream()
            .filter(u -> u != sender)
            .forEach(u -> u.receive(sender.getName() + ": " + message));
    }
}

// 同事类
public class User {
    private final String name;
    private final ChatMediator mediator;

    public User(String name, ChatMediator mediator) {
        this.name = name;
        this.mediator = mediator;
        mediator.addUser(this);
    }

    public void send(String message) {
        mediator.sendMessage(this, message);
    }

    public void receive(String message) {
        System.out.println(name + " received: " + message);
    }

    public String getName() { return name; }
}

// 使用
ChatMediator chatRoom = new ChatRoom();
User alice = new User("Alice", chatRoom);
User bob = new User("Bob", chatRoom);
User charlie = new User("Charlie", chatRoom);

alice.send("Hello everyone!");
// Bob received: Alice: Hello everyone!
// Charlie received: Alice: Hello everyone!
```

**应用场景：**
- 聊天室
- 机场调度系统
- MVC中的Controller（View和Model之间的中介者）
- Java Timer中的调度器

**中介者 vs 观察者：**
| 维度 | 中介者 | 观察者 |
|------|--------|--------|
| 交互方式 | 集中管理 | 广播通知 |
| 耦合度 | 同事类之间零耦合 | 观察者与被观察者有耦合 |
| 控制流 | 中介者控制 | 被观察者触发 |

```java
    }
}

// 抽象
public abstract class Shape {
    protected DrawAPI drawAPI;

    protected Shape(DrawAPI drawAPI) {
        this.drawAPI = drawAPI;
    }

    public abstract void draw();
}

// 具体抽象
public class Circle extends Shape {
    private int x, y, radius;

    public Circle(int x, int y, int radius, DrawAPI drawAPI) {
        super(drawAPI);
        this.x = x;
        this.y = y;
        this.radius = radius;
    }

    @Override
    public void draw() {
        drawAPI.drawCircle(radius, x, y);
    }
}

// 使用
Shape redCircle = new Circle(100, 100, 10, new RedCircle());
Shape blueCircle = new Circle(200, 200, 20, new BlueCircle());
redCircle.draw();
blueCircle.draw();
```


### Q20: 备忘录模式是什么？如何实现？ ⭐⭐

**答案：**

**备忘录模式**在不破坏封装性的前提下，捕获一个对象的内部状态，并在该对象之外保存这个状态，以便以后恢复。

```java
// 备忘录
public class Memento {
    private final String state;

    public Memento(String state) {
        this.state = state;
    }

    public String getState() { return state; }
}

// 发起人
public class Originator {
    private String state;

    public void setState(String state) {
        this.state = state;
    }

    public String getState() { return state; }

    public Memento saveState() {
        return new Memento(state);
    }

    public void restoreState(Memento memento) {
        this.state = memento.getState();
    }
}

// 管理者
public class Caretaker {
    private final Stack<Memento> history = new Stack<>();

    public void save(Memento memento) {
        history.push(memento);
    }

    public Memento undo() {
        return history.isEmpty() ? null : history.pop();
    }
}

// 使用
Originator originator = new Originator();
Caretaker caretaker = new Caretaker();

originator.setState("State 1");
caretaker.save(originator.saveState());

originator.setState("State 2");
caretaker.save(originator.saveState());

originator.setState("State 3");
System.out.println(originator.getState());  // State 3

originator.restoreState(caretaker.undo());
System.out.println(originator.getState());  // State 2

originator.restoreState(caretaker.undo());
System.out.println(originator.getState());  // State 1
```

**实际应用：**
- 编辑器的撤销/重做
- 游戏存档
- 数据库事务回滚
- 版本控制（Git）

### Q21: 设计模式六大原则是什么？ ⭐⭐

**答案：**

**1. 单一职责原则（Single Responsibility Principle, SRP）**
- 一个类应该只有一个引起它变化的原因
- 一个类只负责一项职责

```java
// 违反SRP
public class UserService {
    public void createUser() { /* ... */ }
    public void sendEmail() { /* ... */ }  // 应该分离
}

// 遵循SRP
public class UserService {
    public void createUser() { /* ... */ }
}
public class EmailService {
    public void sendEmail() { /* ... */ }
}
```

**2. 开闭原则（Open/Closed Principle, OCP）**
- 对扩展开放，对修改关闭
- 通过抽象和接口实现

```java
// 违反OCP：新增支付方式需要修改方法
public double calculate(String type) {
    if ("alipay".equals(type)) return 0.01;
    if ("wechat".equals(type)) return 0.005;
    // 新增支付方式需要修改这里
}

// 遵循OCP：策略模式
public interface PaymentStrategy {
    double getFee();
}
```

**3. 里氏替换原则（Liskov Substitution Principle, LSP）**
- 子类对象能够替换其父类对象而不影响程序正确性
- 子类不应重写父类已实现的方法

```java
// 违反LSP
class Rectangle {
    void setWidth(int w) { /* ... */ }
    void setHeight(int h) { /* ... */ }
}
class Square extends Rectangle {
    // 正方形不应该继承矩形（行为不一致）
}
```

**4. 接口隔离原则（Interface Segregation Principle, ISP）**
- 客户端不应依赖它不需要的接口
- 接口应该细粒度

```java
// 违反ISP
public interface Animal {
    void fly();
    void swim();
    void run();
}

// 遵循ISP
public interface Flyable { void fly(); }
public interface Swimmable { void swim(); }
public interface Runnable { void run(); }
```

**5. 依赖倒置原则（Dependency Inversion Principle, DIP）**
- 高层模块不应依赖低层模块，两者都应依赖抽象
- 抽象不应依赖细节，细节应依赖抽象

```java
// 违反DIP
public class UserService {
    private MySQLUserDao userDao = new MySQLUserDao();  // 依赖具体实现
}

// 遵循DIP
public class UserService {
    private final UserDao userDao;  // 依赖抽象
    public UserService(UserDao userDao) {
        this.userDao = userDao;
    }
}
```

**6. 迪米特法则（Law of Demeter, LoD / 最少知道原则）**
- 一个对象应该对其他对象保持最少的了解
- 只与直接的朋友通信

```java
// 违反LoD
user.getDepartment().getManager().getName();  // 链式调用过深

// 遵循LoD
user.getManagerName();  // 封装内部细节
```

### Q23: 创建型设计模式有哪些？分别适用于什么场景？ ⭐⭐

**答案：**

创建型模式关注对象的创建机制，共5种：

| 模式 | 核心思想 | 适用场景 |
|------|---------|---------|
| **单例模式** | 确保只有一个实例 | 配置类、连接池、线程池 |
| **工厂方法** | 子类决定实例化哪个类 | 不同类型的对象创建 |
| **抽象工厂** | 创建一族相关对象 | 跨平台UI、数据库驱动 |
| **建造者模式** | 分步构建复杂对象 | 复杂对象的构建（SQL、HTTP请求） |
| **原型模式** | 通过拷贝创建对象 | 创建成本高的对象 |

**选择指南：**
- 只需要一个实例 → **单例模式**
- 需要灵活创建不同类型 → **工厂方法**
- 需要创建一族相关产品 → **抽象工厂**
- 对象有很多可选参数 → **建造者模式**
- 创建对象成本高 → **原型模式**

### Q26: 访问者模式是什么？如何使用？ ⭐⭐⭐

**答案：**

**访问者模式**表示一个作用于某对象结构中的各元素的操作，使你可以在不改变各元素的类的前提下定义作用于这些元素的新操作。

```java
// 访问者接口
public interface Visitor {
    void visit(ConcreteElementA element);
    void visit(ConcreteElementB element);
}

// 元素接口
public interface Element {
    void accept(Visitor visitor);
}

// 具体元素
public class ConcreteElementA implements Element {
    @Override
    public void accept(Visitor visitor) {
        visitor.visit(this);  // 双分派
    }

    public String operationA() { return "A"; }
}

public class ConcreteElementB implements Element {
    @Override
    public void accept(Visitor visitor) {
        visitor.visit(this);
    }

    public String operationB() { return "B"; }
}

// 具体访问者
public class ConcreteVisitor implements Visitor {
    @Override
    public void visit(ConcreteElementA element) {
        System.out.println("Processing " + element.operationA());
    }

    @Override
    public void visit(ConcreteElementB element) {
        System.out.println("Processing " + element.operationB());
    }
}

// 使用
List<Element> elements = List.of(new ConcreteElementA(), new ConcreteElementB());
Visitor visitor = new ConcreteVisitor();
elements.forEach(e -> e.accept(visitor));
```

**适用场景：**
- 对象结构稳定，但需要频繁添加新操作
- 编译器AST遍历
- XML/JSON文档处理
- 报表生成（同一数据多种报表格式）

**缺点：**
- 增加新的元素类型困难（需要修改所有访问者）
- 违反了依赖倒置原则（访问者依赖具体元素类）

### Q27: 解释器模式是什么？ ⭐⭐⭐

**答案：**

**解释器模式**给定一种语言，定义它的文法的一种表示，并定义一个解释器，这个解释器使用该表示来解释语言中的句子。

```java
// 抽象表达式
public interface Expression {
    int interpret();
}

// 终结符表达式 - 数字
public class NumberExpression implements Expression {
    private final int number;

    public NumberExpression(int number) {
        this.number = number;
    }

    @Override
    public int interpret() {
        return number;
    }
}

// 非终结符表达式 - 加法
public class AddExpression implements Expression {
    private final Expression left;
    private final Expression right;

    public AddExpression(Expression left, Expression right) {
        this.left = left;
        this.right = right;
    }

    @Override
    public int interpret() {
        return left.interpret() + right.interpret();
    }
}

// 非终结符表达式 - 减法
public class SubtractExpression implements Expression {
    private final Expression left;
    private final Expression right;

    public SubtractExpression(Expression left, Expression right) {
        this.left = left;
        this.right = right;
    }

    @Override
    public int interpret() {
        return left.interpret() - right.interpret();
    }
}

// 使用：构建表达式树 10 + 5 - 3
Expression expression = new SubtractExpression(
    new AddExpression(new NumberExpression(10), new NumberExpression(5)),
    new NumberExpression(3)
);
System.out.println(expression.interpret());  // 12
```

**实际应用：**
- 正则表达式引擎
- SQL解析器
- 表达式计算引擎（如Spring Expression Language）
- 模板引擎（如Freemarker、Thymeleaf）
- 规则引擎（如Drools）

### Q28: 如何在实际项目中选择合适的设计模式？ ⭐⭐

**答案：**

**选择设计模式的原则：**

**1. 明确问题**
- 需要创建对象？→ 创建型模式
- 需要组合对象？→ 结构型模式
- 需要管理行为？→ 行为型模式

**2. 常见场景映射**

| 场景 | 推荐模式 |
|------|---------|
| 消除大量if-else | 策略模式 + 工厂模式 |
| 需要在不修改源码的情况下增强功能 | 装饰器模式 / 代理模式 |
| 对象创建过程复杂 | 建造者模式 / 工厂模式 |
| 需要解耦发送者和接收者 | 观察者模式 / 中介者模式 |
| 需要遍历复杂结构 | 组合模式 + 迭代器模式 |
| 需要保存和恢复状态 | 备忘录模式 |
| 需要控制访问权限 | 代理模式 |
| 接口不兼容 | 适配器模式 |
| 需要简化复杂子系统 | 外观模式 |

**3. 避免过度设计**
- 不要为了使用设计模式而使用设计模式
- 简单场景用简单方案
- 遵循KISS原则（Keep It Simple, Stupid）
- YAGNI原则（You Aren't Gonna Need It）

**4. 重构时引入**
- 先写简单的代码
- 当代码出现"坏味道"时再重构引入设计模式
- 例如：大量if-else → 策略模式

### Q30: 设计模式面试中常见的问题总结 ⭐⭐⭐

**答案：**

**高频面试问题：**

**1. 单例模式DCL为什么需要volatile？**
防止`new Singleton()`的指令重排序。`new`操作分三步：分配内存、初始化对象、引用指向内存。步骤2和3可能重排序，导致其他线程拿到未初始化的对象。

**2. JDK动态代理和CGLIB的区别？**
JDK代理基于接口，CGLIB基于继承。Spring AOP默认优先使用JDK代理，目标类没有接口时使用CGLIB。

**3. 策略模式和工厂模式的区别？**
策略模式关注算法的替换，工厂模式关注对象的创建。两者经常配合使用：工厂负责创建策略对象。

**4. 装饰器模式和代理模式的区别？**
装饰器关注增强功能（客户端知道装饰器的存在），代理关注控制访问（客户端不知道代理的存在）。

**5. Spring中Bean的作用域有哪些？**
singleton（默认）、prototype、request、session、application、websocket。

**6. 如何在Spring中实现策略模式？**
定义策略接口 + 多个实现类（@Component） + 策略工厂（通过List注入所有策略，Map缓存）。

**7. 观察者模式和发布-订阅模式的区别？**
- 观察者模式：被观察者直接通知观察者（同步）
- 发布-订阅模式：通过消息中间件解耦（异步），发布者和订阅者互不知道对方

**8. 适配器模式和装饰器模式的区别？**
- 适配器：接口转换，不改变功能
- 装饰器：增加功能，接口不变

**9. 什么是开闭原则？如何实现？**
对扩展开放，对修改关闭。通过抽象和接口实现，新增功能通过扩展而非修改现有代码。

**10. 设计模式在实际项目中的应用举例？**
- 支付系统：策略模式（多种支付方式）
- 权限系统：责任链模式（多级审批）
- 日志系统：装饰器模式（添加日志格式、过滤等）
- 消息系统：观察者模式（事件通知）
- 配置管理：单例模式（全局配置）
