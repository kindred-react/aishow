# 01 云计算基础概念

## 1.1 什么是云计算

云计算（Cloud Computing）是一种通过互联网按需提供计算资源的模式。用户无需购买和维护物理硬件，而是向云服务提供商租用计算能力、存储和网络等资源，并按实际使用量付费。

根据美国国家标准与技术研究院（NIST）的定义，云计算具备以下**五大核心特征**：

1. **按需自助服务（On-Demand Self-Service）**：用户可以在无需人工干预的情况下，自行申请和配置计算资源，如服务器实例、存储空间等。例如通过 AWS 控制台一键创建 EC2 实例。
2. **广泛的网络访问（Broad Network Access）**：资源通过标准网络机制（通常是互联网）提供，支持各种终端设备（手机、平板、笔记本等）访问。
3. **资源池化（Resource Pooling）**：云厂商将多租户的物理和虚拟资源（CPU、内存、存储、网络带宽）统一池化管理，按需动态分配。用户通常不知道资源的精确物理位置。
4. **快速弹性（Rapid Elasticity）**：资源可以快速、弹性地伸缩（自动扩缩容），以应对业务流量的波动。对用户而言，可用的资源似乎无限。
5. **可计量服务（Measured Service）**：资源的使用情况可被监控、控制和报告，实现透明计费。用户只为实际使用的资源付费（按量付费）。

## 1.2 云服务模型

### 1.2.1 IaaS（基础设施即服务）

IaaS（Infrastructure as a Service）提供最基础的虚拟化计算资源，包括虚拟机、虚拟网络、存储等。用户需要自行管理操作系统、中间件、运行时环境和应用程序。

**代表产品**：
- AWS EC2（Elastic Compute Cloud）
- 阿里云 ECS（Elastic Compute Service）
- Azure Virtual Machines
- GCP Compute Engine

**核心特点**：
- 用户拥有对操作系统及以上层级的完全控制权
- 云厂商负责管理底层物理硬件、网络设备、虚拟化层和数据中心设施
- 灵活性最高，但运维负担也最重
- 适合需要完全控制基础设施的场景，如自定义内核模块、特殊运行时需求

### 1.2.2 PaaS（平台即服务）

PaaS（Platform as a Service）在 IaaS 之上提供了运行时环境和中间件，用户只需关注应用程序代码和数据，无需管理操作系统、运行时和中间件。

**代表产品**：
- AWS Elastic Beanstalk、AWS Lambda（Serverless PaaS）
- Heroku
- Google App Engine
- 阿里云 SAE（Serverless App Engine）

**核心特点**：
- 用户只需上传代码和配置，平台自动处理部署、扩缩容和运维
- 平台负责操作系统、运行时环境、中间件的安装和维护
- 开发效率高，适合快速迭代和部署
- 灵活性中等，受限于平台支持的运行时和框架

### 1.2.3 SaaS（软件即服务）

SaaS（Software as a Service）提供完整的、可直接使用的应用程序，用户通过浏览器或客户端访问，无需管理任何基础设施。

**代表产品**：
- Gmail、Google Workspace
- Salesforce（CRM）
- Notion、飞书、钉钉
- Microsoft 365

**核心特点**：
- 开箱即用，用户无需安装、配置或维护任何软件和硬件
- 所有运维工作由 SaaS 提供商承担
- 通常按订阅制（月/年）付费
- 灵活性最低，但使用门槛也最低

### 1.2.4 三种模型对比

| 维度 | IaaS | PaaS | SaaS |
|------|------|------|------|
| 用户管理范围 | OS、运行时、应用、数据 | 应用、数据 | 仅数据 |
| 灵活性 | 最高 | 中等 | 最低 |
| 运维负担 | 最重 | 中等 | 无 |
| 适用场景 | 需要完全控制 | 快速开发部署 | 通用办公工具 |

## 1.3 部署模型

### 1.3.1 公有云

公有云由第三方云服务提供商拥有和运营，通过互联网向公众提供计算资源。多租户共享底层基础设施，每个用户的数据通过虚拟化技术隔离。

**代表厂商**：AWS、Microsoft Azure、Google Cloud Platform（GCP）、阿里云、腾讯云、华为云。

**核心优势**：
- **零前期投入**：无需购买硬件，按需付费，降低 CapEx（资本支出）
- **弹性伸缩**：资源可快速扩缩，应对业务峰值
- **免运维**：用户无需管理物理服务器、网络设备和数据中心
- **全球覆盖**：主流云厂商在全球数十个 Region 部署数据中心，可就近服务用户

**注意事项**：
- 数据存储在第三方基础设施上，需关注合规和数据主权问题
- 多租户环境下需注意资源争抢和隔离安全性

### 1.3.2 私有云

私有云由单个组织独享使用的云计算环境，可以部署在企业自有数据中心或由第三方托管。基础设施专供该组织使用，不与其他组织共享。

**典型场景**：
- **金融行业**：银行、证券等对数据安全和监管合规要求极高
- **政务领域**：政府机构处理敏感数据，需满足等保要求
- **医疗行业**：患者隐私数据保护（如 HIPAA 合规）
- **大型企业**：拥有成熟 IT 团队，需要高度定制化

**核心优势**：
- 数据完全自主可控，满足严格的安全合规要求
- 可根据业务需求深度定制
- 网络延迟可控，适合对延迟敏感的应用

**挑战**：
- 前期投入大（硬件采购、数据中心建设）
- 需要专业运维团队
- 弹性伸缩能力受限于自有资源规模

### 1.3.3 混合云

混合云是将公有云和私有云通过安全的网络连接组合在一起，形成一个统一的 IT 环境。工作负载可以在两者之间灵活迁移。

**核心架构**：
- 使用专线（如 AWS Direct Connect、阿里云高速通道）或 VPN 连接公有云与私有云
- 统一的身份认证和权限管理
- 一致的开发部署流程

**典型应用场景**：
- **弹性扩容**：日常业务运行在私有云，流量高峰时将部分负载溢出到公有云（Cloud Bursting）
- **灾备**：在公有云上建立私有云的灾备副本
- **数据分级**：敏感数据留在私有云，非敏感计算任务放在公有云
- **渐进式迁移**：逐步将部分应用从本地迁移到云端

### 1.3.4 多云策略

多云策略是指同时使用两个或多个公有云服务商的策略，而不是仅依赖单一云厂商。

**核心动机**：
- **避免厂商锁定（Vendor Lock-in）**：不依赖单一厂商，降低议价风险和迁移成本
- **灾备与高可用**：跨云部署关键业务，避免单点故障
- **最佳性价比**：不同云厂商在不同服务上有价格优势，灵活选择
- **合规要求**：某些行业法规要求不能将所有数据放在单一云平台上

**挑战**：
- 运维复杂度显著增加（不同的 API、管理界面、计费模型）
- 跨云网络连接和数据同步的技术难度
- 需要统一的监控、日志和告警体系
- 推荐使用 Terraform、Pulumi 等基础设施即代码（IaC）工具统一管理多云资源

## 1.4 虚拟化技术基础

### 1.4.1 为什么需要虚拟化

虚拟化是云计算的基石技术，其核心价值体现在以下方面：

1. **提高资源利用率**：传统物理服务器 CPU 平均利用率仅 10-20%，通过虚拟化可以在一台物理机上运行多个虚拟机，将利用率提升到 60-80%。
2. **隔离性**：每个虚拟机拥有独立的运行环境，一个虚拟机的故障或安全事件不会影响其他虚拟机。
3. **弹性伸缩**：虚拟机可以在几分钟内创建或销毁，实现资源的动态分配和回收。
4. **降低成本**：减少物理服务器数量，降低硬件采购、机房空间、电力和散热成本。
5. **快速部署**：通过模板和镜像，新环境可以在秒级到分钟级完成部署。
6. **便于迁移**：虚拟机可以在不同物理主机之间迁移，实现负载均衡和维护升级。

### 1.4.2 虚拟化类型
- **全虚拟化**：硬件完全模拟，Guest OS 无需修改即可运行。性能开销较大，但兼容性最好。
- **半虚拟化**：Guest OS 知道自己运行在虚拟环境中，通过修改内核与 Hypervisor 协作，减少模拟开销。性能优于全虚拟化，但需修改 Guest OS。
- **硬件辅助虚拟化**：利用 CPU 硬件扩展（Intel VT-x / AMD-V）直接支持虚拟化操作，大幅提升性能。现代虚拟化方案均基于此技术。

### 1.4.3 网络虚拟化

网络虚拟化将物理网络资源抽象为逻辑网络，实现网络资源的灵活分配和管理。

**核心技术**：
- **VLAN（Virtual LAN）**：基于 IEEE 802.1Q 标准，在二层网络中划分逻辑广播域，最多支持 4094 个 VLAN。适用于小型虚拟化环境，但无法满足大规模云环境需求。
- **VXLAN（Virtual eXtensible LAN）**：将二层以太帧封装在 UDP 报文中（MAC-in-UDP），通过三层网络传输，支持约 1600 万个虚拟网络段。是现代云环境（如 OpenStack、K8s）的主流网络虚拟化方案。
- **SDN（Software Defined Networking，软件定义网络）**：将网络的控制平面和数据平面分离，通过集中式控制器（如 OpenDaylight、ONOS）编程管理网络。实现网络配置的自动化和灵活调度。

### 1.4.4 存储虚拟化

存储虚拟化将多个物理存储设备抽象为统一的逻辑存储池，屏蔽底层存储的物理差异。

**主要类型**：
- **SAN（Storage Area Network）**：提供块级别的存储访问，通过 Fibre Channel 或 iSCSI 协议连接。性能高、延迟低，适合数据库等 I/O 密集型应用。代表产品：EMC VNX、NetApp AFF。
- **NAS（Network Attached Storage）**：提供文件级别的存储访问，通过 NFS/SMB 协议共享。配置简单，适合文件共享场景。代表产品：NetApp FAS、Synology。
- **分布式存储**：将数据分散存储在多个节点上，通过软件实现数据冗余和一致性。可横向扩展，是云厂商的主流选择。代表方案：Ceph（块/对象/文件统一存储）、GlusterFS、MinIO（对象存储）。

## 1.6 云计算核心概念

### 1.6.1 区域（Region）与可用区（AZ）

- **Region（区域）**：云厂商在全球不同地理位置建设的物理数据中心集群。例如 AWS 在全球有 30+ 个 Region（如 us-east-1 弗吉尼亚、ap-northeast-1 东京），阿里云在国内有华北、华东、华南等多个 Region。选择 Region 时需考虑：用户地理位置（降低延迟）、合规要求（数据主权）、服务可用性（部分服务仅在特定 Region 提供）。
- **Availability Zone（可用区，AZ）**：每个 Region 内包含一个或多个 AZ，每个 AZ 是一个或多个物理数据中心的集合，具有独立的电力、制冷和网络。同一 Region 内不同 AZ 之间通过低延迟（通常 < 2ms）的高速光纤互联。将应用部署在多个 AZ 是实现高可用架构的基本手段，可防止单个数据中心故障导致服务中断。

### 1.6.2 弹性伸缩（Auto Scaling）

弹性伸缩是根据业务负载自动调整计算资源数量的机制。

**核心组件**：
- **监控指标**：CPU 利用率、内存使用率、网络流量、自定义指标等
- **伸缩策略**：
  - **目标追踪（Target Tracking）**：自动调整实例数量以保持指标在目标值附近（如 CPU 保持在 60%）
  - **步进伸缩（Step Scaling）**：基于阈值触发，超过不同阈值执行不同伸缩动作
  - **预测性伸缩（Predictive Scaling）**：基于历史流量模式预测未来需求，提前扩容
- **冷却时间（Cooldown）**：伸缩操作后的等待时间，防止频繁伸缩导致震荡

**典型场景**：电商大促期间流量激增，Auto Scaling 自动增加实例数量；大促结束后自动缩减，节省成本。

### 1.6.3 按需付费 vs 预留实例

| 计费模式 | 说明 | 价格 | 适用场景 |
|---------|------|------|---------|
| **按需付费（On-Demand）** | 按实际使用时长计费，随时创建和释放 | 最高 | 短期、突发或不确定的工作负载 |
| **预留实例（Reserved Instance）** | 提前承诺使用 1 年或 3 年，享受大幅折扣（最高可省 72%） | 较低 | 长期稳定运行的核心业务 |
| **Spot 实例** | 竞价使用闲置计算资源，价格随供需波动，可能被回收 | 最低（可省 90%） | 容错性强的工作负载（如批处理、CI/CD） |
| **节省计划（Savings Plans）** | 承诺每小时消费金额，灵活应用于多种计算实例 | 中等 | 需要灵活性的长期工作负载 |

**最佳实践**：对于生产环境的稳定工作负载，建议使用预留实例或节省计划（可节省 40-72%）；对于开发和测试环境使用按需实例；对于无状态批处理任务可使用 Spot 实例。

## 1.7 面试常见问题

- Q1: IaaS、PaaS、SaaS 的区别是什么？举例说明
  - **参考答案要点**：三者区别在于用户管理的层级不同。IaaS 用户管理 OS 及以上（如 AWS EC2），PaaS 用户只管应用和数据（如 Heroku），SaaS 用户直接使用软件无需管理任何基础设施（如 Gmail）。可以用"披萨即服务"类比来解释：IaaS 是自己在家做披萨（买食材自己做），PaaS 是买冷冻披萨回家烤，SaaS 是去餐厅点披萨直接吃。

- Q2: 公有云和私有云各自的优缺点？
  - **参考答案要点**：公有云优点是零前期投入、弹性好、免运维；缺点是数据安全性和合规性需额外关注。私有云优点是数据自主可控、安全合规、可深度定制；缺点是前期投入大、弹性受限、需专业运维团队。实际中很多企业选择混合云，兼顾两者优势。

- Q3: 什么是多云策略？为什么要用多云？
  - **参考答案要点**：多云策略是同时使用多个公有云服务商。主要动机包括：(1) 避免厂商锁定，降低迁移风险；(2) 灾备和高可用，避免单云故障；(3) 利用不同云厂商的差异化优势（如 GCP 的 AI/ML 能力、AWS 的服务丰富度）；(4) 合规要求。挑战在于运维复杂度增加，需要 IaC 工具（Terraform）统一管理。

- Q4: 解释 Region 和 Availability Zone 的概念
  - **参考答案要点**：Region 是地理上的数据中心集群（如 AWS us-east-1 在弗吉尼亚），选择 Region 主要考虑用户延迟和数据合规。AZ 是 Region 内的独立数据中心，具有独立的电力和网络，AZ 之间低延迟互联。高可用架构通常将应用部署在至少 2 个 AZ 上，防止单 AZ 故障。

- Q5: 虚拟化和容器化的区别是什么？
  - **参考答案要点**：虚拟化（VM）通过 Hypervisor 模拟完整硬件，每个 VM 运行独立的 Guest OS，隔离性强但资源开销大（GB 级镜像、分钟级启动）。容器化共享宿主机内核，只隔离进程，资源开销小（MB 级镜像、秒级启动），密度更高。VM 适合需要不同 OS 或强隔离的场景，容器适合微服务和快速部署场景。
