# Webpack 性能优化

## 一、构建速度优化

### 1. 升级 Webpack 版本
使用最新版本的 Webpack，新版本通常包含性能优化和 bug 修复

### 2. 使用 Loader 时缩小范围
通过 `include` 和 `exclude` 配置，减少 loader 处理的文件数量

```javascript
module: {
  rules: [
    {
      test: /\.js$/,
      include: path.resolve(__dirname, 'src'),
      exclude: /node_modules/,
      use: 'babel-loader'
    }
  ]
}
```

### 3. 开启多进程构建
使用 `thread-loader` 开启多进程打包（happypack 已停止维护，不再推荐）

```javascript
// thread-loader
module: {
  rules: [
    {
      test: /\.js$/,
      use: [
        {
          loader: 'thread-loader',
          options: {
            workers: 4, // 建议为 CPU 核数 - 1
          }
        },
        'babel-loader'
      ]
    }
  ]
}
```

### 4. 开启缓存
使用 Webpack 5 的持久化缓存功能（cache-loader 已过时，不再推荐）

```javascript
module.exports = {
  cache: {
    type: 'filesystem',
    buildDependencies: {
      config: [__filename]
    }
  }
}
```

### 5. 开发环境开启热更新
```javascript
devServer: {
  hot: true
  // hotOnly: true 已在 Webpack 5 中移除，hot: true 即可
}
```

### 6. 合理使用 source-map
开发环境使用 `eval-cheap-module-source-map`，生产环境使用 `hidden-source-map`

```javascript
devtool: process.env.NODE_ENV === 'development' 
  ? 'eval-cheap-module-source-map' 
  : 'hidden-source-map'
```

### 7. 使用 DLL 预编译
> ⚠️ 注意：Webpack 5 推荐使用持久化缓存（filesystem cache）替代 DLL 预编译，DLL 方案在新项目中已不推荐

将不常变化的第三方库预先打包

```javascript
// webpack.dll.config.js
module.exports = {
  entry: {
    vendor: ['react', 'react-dom', 'lodash']
  },
  output: {
    filename: '[name].dll.js',
    library: '[name]_library'
  },
  plugins: [
    new webpack.DllPlugin({
      name: '[name]_library',
      path: path.join(__dirname, '[name]-manifest.json')
    })
  ]
}

// webpack.config.js
plugins: [
  new webpack.DllReferencePlugin({
    manifest: require('./vendor-manifest.json')
  })
]
```

### 8. 使用 externals
将大型库通过 CDN 引入，不打包到 bundle 中

```javascript
externals: {
  react: 'React',
  'react-dom': 'ReactDOM',
  lodash: '_'
}
```

## 二、打包体积优化

### 1. Tree Shaking
移除未使用的代码

```javascript
// production 模式下自动开启
mode: 'production'

// 或手动配置
optimization: {
  usedExports: true,
  sideEffects: false
}
```

### 2. 代码分割
将代码分割成多个 chunk，按需加载

```javascript
optimization: {
  splitChunks: {
    chunks: 'all',
    minSize: 30000,
    maxSize: 0,
    minChunks: 1,
    maxAsyncRequests: 5,
    maxInitialRequests: 3,
    automaticNameDelimiter: '~',
    cacheGroups: {
      vendors: {
        test: /[\\/]node_modules[\\/]/,
        priority: -10
      },
      default: {
        minChunks: 2,
        priority: -20,
        reuseExistingChunk: true
      }
    }
  }
}

// 动态导入
import('./module').then(module => {
  module.default()
})
```

### 3. 压缩代码
使用 TerserPlugin 压缩 JavaScript，CssMinimizerPlugin 压缩 CSS

```javascript
optimization: {
  minimize: true,

      parallel: true,
```

## 六、最佳实践总结

1. **开发环境**：快速构建，使用 eval-cheap-module-source-map，开启 HMR
2. **生产环境**：体积最小，使用 hidden-source-map，开启压缩和 Tree Shaking
3. **合理使用缓存**：利用 Webpack 5 的持久化缓存
4. **按需加载**：使用动态导入和代码分割
5. **定期分析**：使用分析工具监控打包体积和构建速度
6. **升级依赖**：保持 Webpack 和相关插件为最新版本
7. **避免过度优化**：在性能和可维护性之间找到平衡点

## 七、首屏加载优化完整方案

### 7.1 SSR / SSG 方案选择

```
CSR（Client Side Rendering）— SPA 默认
  优点：交互体验好、前后端分离
  缺点：首屏慢、SEO 差

SSR（Server Side Rendering）— Next.js / Nuxt.js
  优点：首屏快、SEO 好
  缺点：服务器压力大、TTFB 可能慢

SSG（Static Site Generation）— 静态站点生成
  优点：CDN 分发、加载最快
  缺点：不适合动态内容

ISR（Incremental Static Regeneration）— 增量静态生成
  优点：兼顾静态性能和动态更新
  缺点：数据可能不是最新的
```

```javascript
// Next.js SSG 示例
export async function getStaticProps() {
  const posts = await fetchPosts();
  return { props: { posts } };
}

// Next.js ISR 示例
export async function getStaticProps() {
  const posts = await fetchPosts();
  return {
    props: { posts },
    revalidate: 60 // 60 秒后重新生成
  };
}
```

## 八、Core Web Vitals 优化

### 8.1 指标定义

| 指标 | 全称 | 定义 | 目标 |
|---|---|---|---|
| **LCP** | Largest Contentful Paint | 最大内容元素渲染时间 | <= 2.5s |
| **INP** | Interaction to Next Paint | 交互到下一次绘制的延迟 | <= 200ms |
| **CLS** | Cumulative Layout Shift | 累积布局偏移 | <= 0.1 |

### 8.2 LCP 优化

### 8.3 INP 优化

```javascript
// INP 优化手段：
// 1. 减少主线程阻塞（长任务拆分）
// 2. 使用 requestIdleCallback 执行低优先级任务
// 3. 使用 Web Worker 处理计算密集型任务
// 4. 减少事件处理器中的工作量
// 5. 使用 debounce/throttle 减少事件触发频率

// 长任务拆分示例
function processLargeArray(items, callback) {
  const CHUNK_SIZE = 50;
  let index = 0;

  function processChunk() {
    const end = Math.min(index + CHUNK_SIZE, items.length);
    for (let i = index; i < end; i++) {
      callback(items[i]);
    }
    index = end;

    if (index < items.length) {
      // 让出主线程
      requestAnimationFrame(processChunk);
    }
  }

  processChunk();
}
```

## 十、虚拟列表完整实现

### 10.1 固定高度虚拟列表

```jsx
function VirtualList({ items, itemHeight = 50, containerHeight = 500 }) {
  const [scrollTop, setScrollTop] = useState(0);

  const totalHeight = items.length * itemHeight;
  const startIndex = Math.floor(scrollTop / itemHeight);
  const endIndex = Math.min(
    startIndex + Math.ceil(containerHeight / itemHeight) + 1,
    items.length
  );

  const visibleItems = items.slice(startIndex, endIndex);
  const offsetY = startIndex * itemHeight;

  return (
    <div
      style={{ height: containerHeight, overflow: 'auto' }}
      onScroll={(e) => setScrollTop(e.target.scrollTop)}
    >
      <div style={{ height: totalHeight, position: 'relative' }}>
        <div style={{ position: 'absolute', top: offsetY, width: '100%' }}>
          {visibleItems.map((item, index) => (
            <div key={startIndex + index} style={{ height: itemHeight }}>
              {item}
            </div>
          ))}
        </div>
      </div>
    </div>
  );
}
```

### 10.2 动态高度虚拟列表

```jsx
function DynamicVirtualList({ items, containerHeight = 500, estimatedItemHeight = 50 }) {
  const [scrollTop, setScrollTop] = useState(0);
  const positions = useRef([]); // 每个元素的位置信息
  const measuredRef = useRef(new Map()); // 已测量的元素高度

  // 初始化预估位置
  if (positions.current.length === 0) {
    positions.current = items.map((_, index) => ({
      height: estimatedItemHeight,
      top: index * estimatedItemHeight,
      bottom: (index + 1) * estimatedItemHeight
    }));
  }

  // 更新已测量元素的位置
  const updatePositions = (index, height) => {
    if (measuredRef.current.has(index)) return;
    measuredRef.current.set(index, height);

    const positionsArr = positions.current;
    positionsArr[index] = {
      height,
      top: index > 0 ? positionsArr[index - 1].bottom : 0,
      bottom: (index > 0 ? positionsArr[index - 1].bottom : 0) + height
    };

    // 更新后续元素的位置
    for (let i = index + 1; i < positionsArr.length; i++) {
      positionsArr[i] = {
        ...positionsArr[i],
        top: positionsArr[i - 1].bottom,
        bottom: positionsArr[i - 1].bottom + positionsArr[i].height
      };
    }
  };

  // 二分查找起始索引
  const findStartIndex = (scrollTop) => {
    const positionsArr = positions.current;
    let low = 0, high = positionsArr.length - 1;
    while (low <= high) {
      const mid = Math.floor((low + high) / 2);
      if (positionsArr[mid].bottom > scrollTop) {
        if (mid === 0 || positionsArr[mid - 1].bottom <= scrollTop) return mid;
        high = mid - 1;
      } else {
        low = mid + 1;
      }
    }
    return 0;
  };

  const startIndex = findStartIndex(scrollTop);
  const totalHeight = positions.current[positions.current.length - 1]?.bottom || 0;

  // 渲染可见区域 + 缓冲区
  const visibleItems = [];
  let currentTop = positions.current[startIndex]?.top || 0;
  let endIndex = startIndex;

  while (currentTop < scrollTop + containerHeight && endIndex < items.length) {
    visibleItems.push({ item: items[endIndex], index: endIndex });
    currentTop = positions.current[endIndex].bottom;
    endIndex++;
  }

  return (
    <div
      style={{ height: containerHeight, overflow: 'auto' }}
      onScroll={(e) => setScrollTop(e.target.scrollTop)}
    >
      <div style={{ height: totalHeight, position: 'relative' }}>
        {visibleItems.map(({ item, index }) => (
          <div
            key={index}
            ref={(el) => {
              if (el) updatePositions(index, el.getBoundingClientRect().height);
            }}
            style={{
              position: 'absolute',
              top: positions.current[index].top,
              width: '100%'
            }}
          >
            {item}
          </div>
        ))}
      </div>
    </div>
  );
}
```

## 十一、React.memo / useMemo / useCallback

### 11.1 使用原则

```jsx
// React.memo — 避免子组件不必要的重新渲染
// 适用：纯展示组件，props 不常变化
const ExpensiveList = React.memo(function ExpensiveList({ items }) {
  return items.map(item => <div key={item.id}>{item.name}</div>);
});

// useMemo — 缓存计算结果
// 适用：计算开销大的操作
function SearchResults({ query, allItems }) {
  const filtered = useMemo(
    () => allItems.filter(item => item.name.includes(query)),
    [query, allItems]
  );
  return <List items={filtered} />;
}

// useCallback — 缓存函数引用
// 适用：传递给子组件的回调函数
function Parent() {
  const [count, setCount] = useState(0);
  const [text, setText] = useState('');

  // bad: 每次渲染创建新函数，导致 Child 重新渲染
  // const handleClick = () => setCount(c => c + 1);

  // good: 缓存函数引用
  const handleClick = useCallback(() => setCount(c => c + 1), []);

  return (
    <>
      <input value={text} onChange={e => setText(e.target.value)} />
      <Child onClick={handleClick} />
    </>
  );
}
```

### 11.2 常见误区

```jsx
// 误区 1：所有组件都用 React.memo
```

## 十二、Web Vitals 监控与上报

### 12.1 数据采集

```javascript
import { onLCP, onINP, onCLS, onFCP, onTTFB } from 'web-vitals';

function reportToAnalytics(metric) {
  const body = JSON.stringify({
    name: metric.name,
    value: metric.value,
    rating: metric.rating, // 'good' | 'needs-improvement' | 'poor'
    navigationType: metric.navigationType,
    url: window.location.href,
    timestamp: Date.now()
  });

  // 使用 sendBeacon 确保页面卸载时也能上报
  if (navigator.sendBeacon) {
    navigator.sendBeacon('/api/vitals', body);
  } else {
    fetch('/api/vitals', { body, method: 'POST', keepalive: true });
  }
}

onLCP(reportToAnalytics);
onINP(reportToAnalytics);
onCLS(reportToAnalytics);
onFCP(reportToAnalytics);
onTTFB(reportToAnalytics);
```
