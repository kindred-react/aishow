### Q1: 移动端有哪些适配方案？各有什么优缺点？ ⭐⭐

**答案：**

**1. rem适配方案**
- 以根元素font-size为基准，所有尺寸用rem表示
- 通过JS动态计算html的font-size

```javascript
// flexible.js 核心逻辑
(function () {
  const docEl = document.documentElement;
  function setRemUnit() {
    // 以375px设计稿为基准，1rem = 100px
    const width = docEl.clientWidth;
    docEl.style.fontSize = (width / 3.75) + 'px';
  }
  setRemUnit();
  window.addEventListener('resize', setRemUnit);
  window.addEventListener('pageshow', setRemUnit);
})();
```

```css
/* 设计稿上100px → 1rem */
.box { width: 3.75rem; height: 2rem; font-size: 0.28rem; }
```

**2. vw/vh适配方案（推荐）**
- 1vw = 视口宽度的1%
- 无需JS，纯CSS实现

```css
/* 设计稿375px，100px = 100/375*100 = 26.667vw */
/* 配合postcss-px-to-viewport插件自动转换 */
.box { width: 26.667vw; }
```

```javascript
// postcss.config.js
module.exports = {
  plugins: {
    'postcss-px-to-viewport': {
      viewportWidth: 375,
      unitPrecision: 5,
      viewportUnit: 'vw',
      minPixelValue: 1
    }
  }
};
```

**3. viewport方案**
- 通过meta标签设置视口
- 配合百分比/flex布局

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
```

**方案对比：**

| 方案 | 优点 | 缺点 |
|------|------|------|
| rem | 兼容性好 | 需要JS，有轻微闪烁 |
| vw/vh | 纯CSS，无闪烁 | 老浏览器兼容性 |
| viewport | 简单 | 精度不够 |

### Q2: 移动端1px问题是什么？有哪些解决方案？ ⭐⭐

**答案：**

**问题原因：** 移动端设备像素比（DPR）通常为2或3，CSS的1px在物理设备上实际占2-3个物理像素，导致边框看起来比设计稿粗。

**解决方案：**

**1. transform: scale（推荐）**
```css
.border-1px {
  position: relative;
}
.border-1px::after {
  content: '';
  position: absolute;
  left: 0;
  bottom: 0;
  width: 100%;
  height: 1px;
  background: #000;
  transform: scaleY(0.5); /* DPR=2时缩放0.5 */
  transform-origin: bottom;
}

/* 动态设置scale */
/* DPR=2: scaleY(0.5), DPR=3: scaleY(0.333) */
```

**2. box-shadow模拟**
```css
.border-1px {
  box-shadow: 0 0 0 0.5px #000;
}
```

**3. border-image + 渐变**
```css
.border-1px {
  border-width: 1px;
  border-style: solid;
  border-image: linear-gradient(to bottom, transparent 50%, #000 50%) 1;
}
```

**4. 媒体查询 + min-device-pixel-ratio**
```css
.border-1px {
  border: 1px solid #000;
}
@media (-webkit-min-device-pixel-ratio: 2) {
  .border-1px { border-width: 0.5px; }
}
@media (-webkit-min-device-pixel-ratio: 3) {
  .border-1px { border-width: 0.333px; }
}
```

**5. viewport + rem（通过缩小整个页面）**
```html
<meta name="viewport" content="width=device-width, initial-scale=0.5">
<!-- 将页面整体缩小，CSS的1px就等于物理1px -->
```

### Q3: 移动端点击延迟300ms问题如何解决？ ⭐⭐

**答案：**

**问题原因：** 移动端Safari为了区分单击和双击缩放，在用户点击后会等待约300ms判断是否为双击。

**解决方案：**

**1. 禁用缩放（最简单）**
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
```

**2. CSS touch-action**
```css
html {
  touch-action: manipulation; /* 禁止双击缩放，保留滚动和捏合缩放 */
}
```

**3. FastClick库**
```javascript
import FastClick from 'fastclick';
FastClick.attach(document.body);
```

**4. pointer事件（现代浏览器）**
```javascript
// 使用pointerdown/pointerup代替click
element.addEventListener('pointerdown', handler);
```

**注意：** 现代浏览器（Chrome 32+、iOS Safari 9.3+）在设置了viewport width=device-width后已不再有300ms延迟。

### Q4: 移动端Touch事件有哪些？如何实现手势操作？ ⭐⭐

**答案：**

**Touch事件：**
```javascript
// touchstart - 手指触摸屏幕
// touchmove - 手指在屏幕上移动
// touchend - 手指离开屏幕
// touchcancel - 触摸被打断（如来电）

element.addEventListener('touchstart', (e) => {
  const touch = e.touches[0]; // 当前所有触摸点
  console.log(touch.clientX, touch.clientY);
  console.log(touch.pageX, touch.pageY);
  console.log(touch.identifier); // 触摸点唯一标识
});

element.addEventListener('touchmove', (e) => {
  e.preventDefault(); // 阻止默认滚动行为
  const touch = e.touches[0];
});

element.addEventListener('touchend', (e) => {
  const touch = e.changedTouches[0]; // 发生变化的触摸点
});
```

**实现滑动手势：**
```javascript
class SwipeDetector {
  constructor(element) {
    this.startX = 0;
    this.startY = 0;
    this.threshold = 50; // 最小滑动距离

    element.addEventListener('touchstart', (e) => {
      this.startX = e.touches[0].clientX;
      this.startY = e.touches[0].clientY;
    });

    element.addEventListener('touchend', (e) => {
      const endX = e.changedTouches[0].clientX;
      const endY = e.changedTouches[0].clientY;
      const diffX = endX - this.startX;
      const diffY = endY - this.startY;

      if (Math.abs(diffX) > Math.abs(diffY) && Math.abs(diffX) > this.threshold) {
        const direction = diffX > 0 ? 'right' : 'left';
        element.dispatchEvent(new CustomEvent('swipe', { detail: { direction } }));
      }
    });
  }
}

// 使用
new SwipeDetector(document.getElementById('carousel'));
document.getElementById('carousel').addEventListener('swipe', (e) => {
  console.log('Swipe:', e.detail.direction);
});
```

**实现捏合缩放：**
```javascript
let initialDistance = 0;
let currentScale = 1;

element.addEventListener('touchstart', (e) => {
  if (e.touches.length === 2) {
    const dx = e.touches[0].clientX - e.touches[1].clientX;
    const dy = e.touches[0].clientY - e.touches[1].clientY;
    initialDistance = Math.hypot(dx, dy);
  }
});

element.addEventListener('touchmove', (e) => {
  if (e.touches.length === 2) {
    const dx = e.touches[0].clientX - e.touches[1].clientX;
    const dy = e.touches[0].clientY - e.touches[1].clientY;
    const distance = Math.hypot(dx, dy);
    currentScale = distance / initialDistance;
    element.style.transform = `scale(${currentScale})`;
  }
});
```

### Q6: 什么是PWA？核心特性有哪些？ ⭐⭐

**答案：**

**PWA（Progressive Web App，渐进式Web应用）** 是使用现代Web技术构建的、具有原生应用体验的Web应用。

**核心特性：**
1. **可安装**：可以添加到主屏幕
2. **离线可用**：通过Service Worker缓存资源
3. **推送通知**：支持服务端推送
4. **响应式**：适配各种设备
5. **安全**：必须使用HTTPS

**Service Worker：**
```javascript
// 注册Service Worker
if ('serviceWorker' in navigator) {
  navigator.serviceWorker.register('/sw.js')
    .then(registration => console.log('SW registered'))
    .catch(error => console.log('SW failed:', error));
}

// sw.js - 缓存策略
const CACHE_NAME = 'my-app-v1';
const urlsToCache = [
  '/',
  '/index.html',
  '/styles.css',
  '/app.js'
];

// 安装：预缓存资源
self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME)
      .then(cache => cache.addAll(urlsToCache))
  );
});

// 激活：清理旧缓存
self.addEventListener('activate', (event) => {
  event.waitUntil(
    caches.keys().then(keys => Promise.all(
      keys.filter(key => key !== CACHE_NAME).map(key => caches.delete(key))
    ))
  );
});

// 拦截请求：缓存优先策略
self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request)
      .then(response => response || fetch(event.request))
  );
});
```

```json
  "icons": [
    { "src": "/icons/icon-192.png", "sizes": "192x192", "type": "image/png" },
```

### Q7: Service Worker的生命周期是什么？ ⭐⭐⭐

**答案：**

Service Worker的生命周期分为三个阶段：**安装（Install）→ 等待（Waiting）→ 激活（Active）**。

```
注册 → 安装(Install) → 等待(Waiting) → 激活(Activate) → 控制页面(Fetch)
         ↓                ↓               ↓
      预缓存资源      等待旧SW释放     清理旧缓存
```

**详细流程：**

```javascript
// 1. 注册
navigator.serviceWorker.register('/sw.js', { scope: '/' });

// 2. 安装阶段
self.addEventListener('install', (event) => {
  console.log('SW installing');
  // skipWaiting: 跳过等待，立即激活
  self.skipWaiting();

  event.waitUntil(
    caches.open('cache-v1').then(cache => cache.addAll([
      '/', '/index.html', '/styles.css', '/app.js'
    ]))
  );
});

// 3. 等待阶段
// 新SW安装完成但旧SW仍在控制页面
// 旧SW关闭后新SW才能激活

// 4. 激活阶段
self.addEventListener('activate', (event) => {
  console.log('SW activating');
  // clients.claim: 立即控制所有页面
  self.clients.claim();

  event.waitUntil(
    caches.keys().then(keys => Promise.all(
      keys.filter(key => key !== 'cache-v1').map(key => caches.delete(key))
    ))
  );
});

// 5. Fetch阶段
self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request)
      .then(cached => cached || fetch(event.request))
  );
});
```

**缓存策略：**

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| Cache First | 优先缓存，缓存没有再请求 | 静态资源 |
| Network First | 优先网络，失败再读缓存 | API数据 |
| Stale While Revalidate | 先返回缓存，后台更新 | 频繁更新的资源 |
| Cache Only | 只读缓存 | 预缓存的资源 |
| Network Only | 只请求网络 | 实时数据 |

### Q8: 微信小程序的双线程模型是什么？ ⭐⭐⭐

**答案：**

微信小程序采用**双线程模型**架构：

**1. 逻辑层（AppService）**
- 运行在JSCore（iOS）或V8（Android）中
- 负责执行JavaScript逻辑
- 处理数据绑定、事件处理、API调用
- 无法直接操作DOM

**2. 渲染层（WebView）**
- 运行在WebView中
- 负责页面渲染（WXML → Native组件）
- 有多个WebView（每个页面一个）

**3. 系统层（Native）**
- 提供原生能力（相机、定位、支付等）
- 负责两层之间的通信

**通信机制：**
```
逻辑层 ←→ Native（JSBridge） ←→ 渲染层

// 逻辑层通过setData触发渲染
this.setData({ text: 'hello' });
// → 通过JSBridge传递到Native
// → Native传递到渲染层WebView
// → 渲染层更新视图

// 用户交互事件
// 渲染层捕获事件 → Native → 逻辑层
```

**双线程模型的优势：**
1. 安全：逻辑层无法直接操作DOM
2. 性能：逻辑层不会阻塞渲染层
3. 多页面：每个页面独立WebView

**双线程模型的劣势：**
1. setData性能瓶颈：数据需要序列化传输
2. 通信延迟：跨线程通信有开销
3. 调试复杂：两个线程需要分别调试

### Q11: 小程序路由与页面栈有什么特点？ ⭐⭐

**答案：**

**路由方式：**

| API | 说明 | 页面栈变化 |
|-----|------|-----------|
| `wx.navigateTo` | 保留当前页，跳转到新页面 | +1 |
| `wx.redirectTo` | 关闭当前页，跳转到新页面 | 不变 |
| `wx.switchTab` | 跳转到TabBar页面 | 清空非TabBar页面 |
| `wx.reLaunch` | 关闭所有页面，打开新页面 | 清空 |
| `wx.navigateBack` | 返回上一页 | -1 |
| `wx.navigateBack({ delta: n })` | 返回前n页 | -n |

**页面栈限制：**
- 页面栈最多10层
- 超过10层后`wx.navigateTo`会失败
- 解决方案：使用`wx.redirectTo`或`wx.reLaunch`

**获取页面栈：**
```javascript
const pages = getCurrentPages();
console.log(pages.length); // 当前页面栈深度
console.log(pages[pages.length - 1]); // 当前页面实例
console.log(pages[pages.length - 2]); // 上一个页面实例

// 修改上一页数据
const prevPage = pages[pages.length - 2];
prevPage.setData({ key: 'value' });
```

**TabBar页面：**
- TabBar页面不能使用`wx.navigateTo`跳转
- TabBar页面必须使用`wx.switchTab`
- TabBar页面最多5个（小程序基础库2.5.0后支持自定义TabBar）

### Q12: 小程序性能优化有哪些方案？ ⭐⭐⭐

**答案：**

**1. setData优化（最重要）**

```javascript
// 不好的做法：频繁setData
for (let i = 0; i < 100; i++) {
  this.setData({ list: [...this.data.list, item] });
}

// 好的做法：批量更新
const newList = [...this.data.list, ...items];
this.setData({ list: newList });

// 不好的做法：传输大量不需要的数据
this.setData({ list: this.data.list }); // 整个list传输

// 好的做法：使用路径更新
this.setData({ 'list[0].name': 'new name' });

// 不好的做法：传输不需要渲染的数据
this.setData({ userInfo: { ...this.data.userInfo, token: 'xxx' } });

// 好的做法：非渲染数据不通过setData
this._token = 'xxx'; // 存储在this上，不触发渲染
```

**2. 分包加载**
```json
{
  "subPackages": [
    {
      "root": "packageA",
      "pages": ["pages/pageA/pageA"]
    },
    {
      "root": "packageB",
      "pages": ["pages/pageB/pageB"]
    }
```

### Q13: React Native的Bridge机制是什么？ ⭐⭐⭐

**答案：**

**传统Bridge架构：**
```
JavaScript Thread ←→ Bridge ←→ Native Thread

// JS调用Native
JS → JSON序列化 → Bridge → 反序列化 → Native执行 → 结果序列化 → Bridge → 反序列化 → JS

// 问题：
// 1. 序列化/反序列化开销大
// 2. 跨线程通信异步，有延迟
// 3. 无法同步调用
```

**JSI（JavaScript Interface）新架构：**
```
JavaScript ←→ JSI ←→ C++ TurboModules / Fabric

// JSI优势：
// 1. 直接内存引用，无需序列化
// 2. 支持同步调用
// 3. C++层直接暴露给JS
// 4. 懒加载模块
```

**通信方式：**

```javascript
// 1. Native Module（JS调用原生）
import { NativeModules } from 'react-native';
const { ToastExample } = NativeModules;
ToastExample.show('Hello', ToastExample.SHORT);

// 2. Native Event（原生通知JS）
import { NativeEventEmitter, NativeModules } from 'react-native';
const emitter = new NativeEventEmitter(NativeModules.EventEmitter);
emitter.addListener('EventName', (event) => {
  console.log(event);
});
```

```javascript
  constructor() {
    this.callbacks = {};
    this.uniqueId = 0;
  }

  callNative(handlerName, data, callback) {
    const id = ++this.uniqueId;
    if (callback) this.callbacks[id] = callback;

    const message = JSON.stringify({ id, handlerName, data });
    // 根据平台选择通信方式

  }

      delete this.callbacks[id];
    }
```

### Q17: 小程序的setData为什么有性能问题？如何优化？ ⭐⭐⭐

**答案：**

**setData性能问题的原因：**

1. **跨线程通信**：逻辑层（JSCore）通过Native桥接将数据传递到渲染层（WebView），数据需要序列化为字符串传输
2. **全量传输**：即使只修改一个字段，整个data对象都会被序列化传输
3. **触发渲染**：每次setData都会触发渲染层的diff和重渲染

**优化方案：**

```javascript
// 1. 路径更新（只传输修改的路径）
this.setData({ 'list[0].name': 'new name' });
// 而不是
this.setData({ list: this.data.list }); // 传输整个list

// 2. 合并更新（减少setData调用次数）
// 不好的做法
this.setData({ a: 1 });
this.setData({ b: 2 });
this.setData({ c: 3 });

// 好的做法
this.setData({ a: 1, b: 2, c: 3 });

// 3. 避免传输不必要的数据
// 不好的做法
this.setData({ list: this.data.list }); // list没有变化也传输了

// 4. 非渲染数据不要放在data中
// data中的数据变化会触发渲染
// 非渲染数据放在this上
Page({
  onLoad() {
    this._timer = null; // 非渲染数据
    this._cache = {};   // 非渲染数据
  }
});

// 5. 使用节流/防抖
// 频繁更新的场景（如滚动位置）使用节流
onPageScroll: throttle(function(e) {
  this.setData({ scrollTop: e.scrollTop });
}, 100)

// 6. 使用WXS处理频繁计算
// WXS运行在渲染层，无需跨线程通信
```

### Q20: 小程序如何实现登录流程？ ⭐⭐⭐

**答案：**

```
1. 前端调用 wx.login() 获取临时登录凭证 code
2. 前端将 code 发送到后端
3. 后端用 code + appid + secret 调用微信接口换取 openid 和 session_key
4. 后端生成自定义登录态 token（如JWT）
5. 后端返回 token 给前端
6. 前端存储 token，后续请求携带 token
```

```javascript
// 前端代码
Page({
  async onLaunch() {
    // 1. 检查本地是否有token
    const token = wx.getStorageSync('token');
    if (token) {
      this.checkSession(token);
      return;
    }

    // 2. 登录
    this.login();
  },

  async login() {
    // 获取code
    const { code } = await wx.login();

    // 发送到后端
    const res = await wx.request({
      url: 'https://api.example.com/login',
      method: 'POST',
      data: { code }
    });

    // 存储token
    wx.setStorageSync('token', res.data.token);
    wx.setStorageSync('openid', res.data.openid);
  },

  async checkSession(token) {
    // 检查登录态是否过期
    try {
      await wx.checkSession();
      // session_key未过期
    } catch {
      // session_key已过期，重新登录
      this.login();
    }
  }
});
```

```javascript
// 后端代码（Node.js）
const axios = require('axios');

app.post('/login', async (req, res) => {
  const { code } = req.body;

  // 用code换取openid和session_key

    params: {
      appid: APP_ID,
      secret: APP_SECRET,
      js_code: code,
      grant_type: 'authorization_code'
    }
  });

  const { openid, session_key } = wxRes.data;

  // 生成自定义token

  res.json({ token, openid });

    });
```

```javascript
});
```

### Q28: 小程序的本地存储有哪些？和Web Storage有什么区别？ ⭐⭐

**答案：**

**小程序存储API：**
```javascript
// 同步API
wx.setStorageSync('key', 'value');
const value = wx.getStorageSync('key');
wx.removeStorageSync('key');
wx.clearStorageSync();

// 异步API
wx.setStorage({ key: 'key', data: 'value' });
wx.getStorage({ key: 'key', success(res) { console.log(res.data); } });
wx.removeStorage({ key: 'key' });
wx.clearStorage();

// 获取存储信息
wx.getStorageInfo({
  success(res) {
    console.log(res.keys);     // 所有key
    console.log(res.currentSize); // 当前占用空间（KB）
    console.log(res.limitSize);   // 限制空间（KB）
  }
});
```

**与Web Storage的区别：**

| 特性 | 小程序存储 | Web Storage |
|------|-----------|-------------|
| 同步API | 支持（wx.setStorageSync / wx.getStorageSync） | 不支持（localStorage是同步的） |
| 容量 | 单个key上限1MB，总上限10MB | 约5-10MB |
| 类型限制 | 只能存储可序列化的数据 | 只能存储字符串 |
| 跨域 | 同小程序内共享 | 同源共享 |
| API | wx.setStorage等 | localStorage.setItem等 |

**答案：**

```javascript
Page({
  onShareAppMessage() {
    return {
      title: '分享标题',
      path: '/pages/detail/index?id=123',
      imageUrl: '/images/share.png', // 分享图片（比例5:4）

    };

```

### Q30: 移动端如何实现骨架屏？ ⭐⭐

**答案：**

**方案一：CSS实现**
```html
<div class="skeleton">
  <div class="skeleton-avatar"></div>
  <div class="skeleton-content">
    <div class="skeleton-title"></div>
    <div class="skeleton-text"></div>
    <div class="skeleton-text short"></div>
  </div>
</div>
```

```css
.skeleton {
  display: flex;
  padding: 16px;
  gap: 12px;
}

.skeleton-avatar {
  width: 48px;
  height: 48px;
  border-radius: 50%;
  background: linear-gradient(90deg, #f0f0f0 25%, #e0e0e0 50%, #f0f0f0 75%);
  background-size: 200% 100%;
  animation: shimmer 1.5s infinite;
}

.skeleton-title {
  height: 20px;
  width: 60%;
  border-radius: 4px;
  background: linear-gradient(90deg, #f0f0f0 25%, #e0e0e0 50%, #f0f0f0 75%);
  background-size: 200% 100%;
  animation: shimmer 1.5s infinite;
  margin-bottom: 8px;
}

.skeleton-text {
  height: 14px;
  width: 100%;
  border-radius: 4px;
  background: linear-gradient(90deg, #f0f0f0 25%, #e0e0e0 50%, #f0f0f0 75%);
  background-size: 200% 100%;
  animation: shimmer 1.5s infinite;
  margin-bottom: 6px;
}

.skeleton-text.short { width: 40%; }

@keyframes shimmer {
  0% { background-position: -200% 0; }
  100% { background-position: 200% 0; }
}
```

**方案二：小程序骨架屏**
```html

  <view class="skeleton-block" style="width:100%;height:30rpx;margin-top:16rpx;"></view>
</view>

<view wx:else>
  <!-- 实际内容 -->
</view>
```
