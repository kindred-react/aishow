# 产品愿景文档

**项目**：抖音关注博主筛选助手
**阶段**：市场规划与调研
**日期**：2024年

---

## 一，产品定位

### 1.1 一句话定位

> **抖音关注博主的"智能筛选与内容加工"工具**

### 1.2 详细定位

建立在抖音现有能力之上（关注关系、博主主页信息、视频标题/文案/标签、收藏/喜欢等），做"更强的关注治理与信息加工层"，帮助用户把注意力分配到更值得的创作者与内容上。

### 1.3 产品类型

| 维度 | 描述 |
|------|------|
| 工具类型 | SaaS工具 / 个人效率应用 |
| 平台形态 | Web App（暂定） |
| 数据来源 | 用户授权导入（不上传抖音数据） |
| 核心能力 | 关注治理 + AI分类 + 内容加工 |

---

## 二、核心价值主张

### 2.1 为用户解决什么问题

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   问题：抖音关注太多，刷不过来，不知道该看谁                   │
│                                                             │
│   解决方案：帮你在不取关的前提下，筛选出最值得看的内容         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 核心价值

| 价值类型 | 描述 | 用户收益 |
|---------|------|---------|
| **减法价值** | 智能分层 + 规则筛选 | 减少无效浏览时间 |
| **加工价值** | 视频摘要 + 要点提炼 | 降低信息获取成本 |
| **习惯价值** | 周报复盘 + 定时提醒 | 建立健康管理习惯 |
| **发现价值** | 同赛道推荐 + 优质发现 | 持续补充高质量信源 |

## 三、产品原则

### 3.1 设计原则

| 原则 | 描述 | 示例 |
|------|------|------|
| **用户控制** | 最终决策权在用户 | 只给建议，不自动取关 |
| **透明可解释** | 所有结论可追溯 | 显示分层原因 |
| **渐进增强** | 先手工，再自动 | MVP全手动，V1开始AI辅助 |
| **隐私优先** | 数据本地优先 | 不上传抖音数据 |

### 3.2 不做什么（Non-Goals）

| 分类 | 描述 |
|------|------|
| 内容搬运 | 不做抖音内容的搬运/下载/二次分发 |
| 算法替代 | 不承诺替代抖音推荐算法 |
| 全平台同步 | 初期不做B站/小红书/微博同步 |
| 自动操作 | 不自动执行关注/取关等账号操作 |

## 四、产品路线图

### 4.1 MVP阶段

**目标**：验证核心假设，跑通手动管理闭环

```
手动导入 → 标签打标 → 优先级分层 → 多维筛选 → 复盘清单 → CSV导出
```

### 4.2 V1阶段

**目标**：减少手动操作，AI辅助决策

```
AI辅助评分 → 规则自动分层 → 用户反馈学习 → 半自动导入
```

### 4.3 V2阶段

**目标**：内容深度加工，降低信息获取成本

```
视频摘要 → 博主聚类 → 知识周报 → 娱乐播放清单
```

### 4.4 V3阶段

**目标**：生态扩展，跨平台统一管理

```
跨平台管理 → 知识策展(Timely/Evergreen) → 优质博主发现
```
