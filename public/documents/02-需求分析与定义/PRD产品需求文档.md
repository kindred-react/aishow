## 1. 背景与问题陈述

### 1.1 问题背景

我在抖音关注的博主数量过多，信息流噪声大，难以高效获取"对我有用的信息"。

现有平台内的分组/收藏/稍后看等能力不足以支持"持续、自动、可解释" 的关注管理与筛选。

### 1.2 产品定位

建立在抖音现有能力之上（关注关系、博主主页信息、视频标题/文案/标签、收藏/喜欢等），做"更强的关注治理与信息加工层"，帮助用户把注意力分配到更值得的创作者与内容上。

---

## 2. 目标（Goals）

### 2.1 核心目标

1. 帮助用户在不取关的前提下，对已关注博主做结构化管理与优先级排序
2. 让用户能快速得到"值得看"的内容与创作者列表，并可随时间动态调整
3. 让筛选规则可配置、可解释、可复用（沉淀为模板）
4. 自动识别异常关注（已注销/不可访问/疑似封禁）并自动处理或提示处理
5. 从博主视频中提取主要信息，降低刷视频的时间成本

### 2.2 次要目标

在"先筛选、再增量优化"的前提下：
- 辅助用户发现并关注优质博主
- 通过行为引导优化抖音用户画像（影响推荐分发）

---

## 3. 非目标（Non-goals）

| 分类 | 描述 |
|------|------|
| 内容搬运 | 不做抖音内容的搬运/下载/二次分发 |
| 算法替代 | 不承诺替代抖音推荐算法；目标是"基于关注关系的个人信息治理" |
| 全平台 | 初期不做全量跨平台（B站/小红书/微博）同步；作为后续扩展 |
| 自动操作 | 不在未授权情况下自动执行关注/取关等账号操作（默认只给建议与待确认清单） |

---

## 4. 目标用户与典型场景

### 4.1 目标用户画像

| 用户类型 | 特征描述 |
|---------|---------|
| 重度抖音用户 | 关注数 300+，每天浏览 30 分钟以上 |
| 有明确学习/信息目标的用户 | 例如产品、运营，投资、健身、英语等 |
| 内容囤积型用户 | 收藏很多但很少回看，缺少体系化整理 |

### 4.2 典型场景（User Stories）

| 场景 | 描述 |
|------|------|
| 分层管理 | 我想把关注的博主按"领域/价值/更新频率/可信度"分层，优先看高价值博主 |
| 发现优质 | 我想找到"最近 30 天持续输出且质量高"的博主，作为重点关注 |
| 清理降权 | 我想识别"重复内容/带货过重/标题党"的博主，降权或加入黑名单 |
| 每周复盘 | 我想每周收到一份清单：本周值得看的人 + 值得取消优先级的人 |

---

## 5. 需求概述（What we build）

### 5.1 产品核心

一个"关注博主管理与筛选"应用，围绕三件事：

```
┌─────────────────────────────────────────────┐
│                 三件事                        │
├─────────────────────────────────────────────┤
│  1) 导入关注列表（手动/半自动）              │
│  2) 给每个博主打标签与评分（规则 + AI 辅助）  │
│  3) 输出可行动结果（分组、优先级、清单、提醒） │
└─────────────────────────────────────────────┘
```

### 5.2 核心思想

不同类型创作者的"看不过来"解决方式不同：

| 创作者类型 | 解决方案 | 关键动作 |
|-----------|---------|---------|
| 优质/知识型创作者 | 内容加工 | 把多条视频压缩成可读的要点与结论，降低信息获取成本 |
| 娱乐/审美/抽象创作者 | 时间与场景管理 | 分类、分组与限时查看，把娱乐放到合适的闲暇时间段 |

## 6. 功能需求详述

### 6.1 关注列表导入

#### 6.1.1 手动导入

- 用户粘贴/录入博主昵称、主页链接、备注

#### 6.1.2 半自动导入（V2）

- 通过用户导出的数据/截图识别（OCR）/分享链接批量导入
- 说明：涉及平台权限与合规，需单独评估

#### 6.1.3 关注状态识别与自动取关

**目标**：识别"已注销/主页不可访问/内容清空/账号异常"等博主，并进行自动取关或进入待确认队列。

**识别维度**：
- 主页访问结果：404/提示"该用户已注销/不存在/被封禁"等状态页
- 基础字段缺失：昵称/头像/作品列表异常为空
- 长期不可访问：连续 N 次检测失败

**处理策略**：
- 自动取关：仅在用户显式开启"自动取关已注销博主"开关后启用
- 默认策略：进入"待确认取关"清单，用户一键确认
- 记录：每次识别与处理写入日志（时间、原因、证据）

**安全与可控**：
- 支持撤销（在应用内恢复为"关注但降权/标记"，或提供重新关注指引）
- 支持白名单（永不自动取关）

---

### 6.2 博主画像（Creator Profile）

每个博主形成一张卡片：

#### 6.2.1 基本信息

| 字段 | 描述 |
|------|------|
| 昵称 | 博主名称 |
| 主页链接 | 抖音主页URL |
| 平台 | 抖音（固定） |
| 头像 | 可选 |
| 简介 | 可选 |

#### 6.2.2 账号状态

| 状态 | 描述 |
|------|------|
| 正常 | 可正常访问 |
| 已注销 | 账号已注销 |
| 不可访问 | 主页无法访问 |
| 疑似异常 | 需要复核 |

#### 6.2.3 分类标签

**L1 分类（用户动机层）**：

| 分类 | 描述 | 示例 |
|------|------|------|
| 知识/学习 | 历史分析、人文讲解、科普、读书、方法论 | 老高小术、所长林超 |
| 资讯/观点 | 商业、科技、热点解读，投资观点 | 商业财经、科技解说 |
| 技能/教程 | 健身训练、英语学习、摄影剪辑、做饭手工 | 健身教程、语言学习 |
| 审美/颜值 | 好看/帅（外貌与氛围为主） | 颜值博主、时尚 |
| 娱乐/抽象 | 搞笑、整活、抽象段子、二创 | 搞笑博主、剧情 |
| 情绪价值 | 治愈、陪伴、共鸣、生活方式 | 生活博主、治愈系 |
| 其他 | 带货导购、直播切片等 | 带货、直播 |

**L2 分类（抖音赛道层）**：
采用抖音的更细赛道名称（美食、健身、数码、汽车、母婴、影视、二次元、三农、剧情等）

**规则**：
- 信息提取模板按 L1 决定
- 推荐与聚合可按 L2 做更精细的分组
- 类目支持多选，但需指定一个"主类目"

#### 6.2.4 内容类型

教程、观点、资讯、日常、带货、直播切片…

#### 6.2.5 关注目的

学习、消遣、情绪、社交、灵感、跟踪行业…

#### 6.2.6 质量评分

| 维度 | 描述 | 评分范围 |
|------|------|---------|
| 信息密度 | 内容信息量 | 1-5 |
| 可信度 | 内容可信程度 | 1-5 |
| 原创度 | 内容原创性 | 1-5 |
| 可操作性 | 内容的可执行性 | 1-5 |
| 重复度 | 与其他博主重复程度 | 1-5 |
| 广告占比 | 带货/广告内容比例 | 1-5 |

#### 6.2.7 更新行为

| 字段 | 描述 |
|------|------|
| 最近更新时间 | 最后检测到更新的时间 |
| 更新频率 | 周更/日更/不稳定/停更 |

#### 6.2.8 风险/负面标记

标题党、低质搬运、强带货、情绪消耗

---

### 6.3 筛选与分层

#### 6.3.1 多维筛选

支持按以下维度筛选：
- 领域（L1/L2分类）
- 评分区间
- 更新频率
- 广告占比
- 是否黑名单

#### 6.3.2 优先级分层

| 层级 | 描述 | 典型用户意图 |
|------|------|-------------|
| S | 必看（高价值） | 每天必刷 |
| A | 常看（稳定价值） | 有空就看 |
| B | 偶尔（补充/娱乐） | 偶尔看看 |
| C | 观察（暂存/可能取关） | 观望中 |
| D | 屏蔽（不推荐） | 不再看 |

#### 6.3.3 规则自动分层

可配置的规则示例：
- 近 30 天更新 ≥ 8 且信息密度 ≥ 4 → A 以上
- 广告占比 ≥ 4 或重复度 ≥ 4 → 降 1-2 级

---

### 6.4 AI 辅助（V1+）

#### 6.4.1 初始评分推荐

- 从用户输入的简介/代表内容摘要中，推荐标签与初始评分
- 解释型输出：为什么推荐/降权（可展示规则命中与证据）
- 支持用户反馈校准：用户上调/下调后，模型/规则权重调整

#### 6.4.2 视频内容摘要（V2）

**前置原则**：先分类，再提取。不同类目对应不同的"主要信息"定义与摘要模板。

**支持范围**：
- 单条视频：标题复述、3-7 条要点、结论/建议（若有）、适用人群
- 博主维度：对最近 N 条视频做聚类，产出"该博主主要讲什么/近期主题变化/高频观点"

**输入来源**：
- 公开可获取的标题/文案/话题标签
- 用户手动粘贴的视频文案/字幕
- 通过转写得到的字幕（若技术和合规允许）

**输出要求**：
- 结构化：要点列表 + 关键术语 + 可执行清单（如适用）
- 可追溯：标注信息来源（标题/文案/字幕/用户输入）
- 可交互：用户可点选"有用/无用"，用于后续筛选与分层

**分类型摘要模板**：

| 类型 | 摘要模板 |
|------|---------|
| 知识/学习 | 核心观点、事实/证据点、时间线/人物关系、延伸关键词 |
| 技能/教程 | 步骤清单、注意事项、适用条件、工具/资料 |
| 资讯/观点 | 结论、论据、立场倾向、可信度提示、反例/争议点 |
| 审美/颜值 | 氛围标签、造型要点、拍摄/剪辑风格、代表作索引 |
| 娱乐/抽象 | 梗点提炼、笑点来源、是否重复、适合的观看场景 |

---

### 6.5 输出与行动

#### 6.5.1 差异化输出（关键）

| 内容类型 | 输出形态 |
|---------|---------|
| 知识/技能/资讯（偏"优质信息"） | 博主周报、视频摘要卡 |
| 审美/娱乐/情绪（偏"娱乐放松"） | 闲暇播放清单、风格标签 |
| 其他（带货等） | 广告占比预警/降权建议 |

#### 6.5.2 周报与清单

| 输出类型 | 描述 | 周期 |
|---------|------|------|
| 今日必看清单 | S/A级创作者今日更新 | 每日 |
| 本周周报 | 本周核心观点/方法、可执行清单 | 每周 |
| 待复核清单 | 长期不更新/广告变多/重复 | 每周 |
| 取关建议 | 不值得关注的博主列表 | 按需 |

#### 6.5.3 导出

支持导出为：
- CSV
- Notion（V2）
- Excel（V2）

---

### 6.6 运营与习惯养成

#### 6.6.1 每周复盘提醒

- 本周新增高价值博主
- 本周降权/屏蔽博主
- 需要确认的博主清单

#### 6.6.2 时间预算模式

- 用户设置每日浏览时间预算（如15分钟）
- 仅推荐 S/A 级创作者内容

## 8. 数据结构

### 8.1 Creator（创作者）

```typescript
interface Creator {
  creator_id: string;           // 唯一标识
  name: string;                 // 昵称
  platform: string;             // 平台（抖音）
  profile_url: string;           // 主页链接
  avatar?: string;              // 头像URL
  bio?: string;                 // 简介
  tags: string[];              // L1/L2标签（多选）
  primary_tag: string;           // 主类目
  priority: 'S' | 'A' | 'B' | 'C' | 'D';  // 优先级
  scores: {                     // 评分
    info_density: number;       // 信息密度 1-5
    credibility: number;       // 可信度 1-5
    originality: number;       // 原创度 1-5
    actionability: number;      // 可操作性 1-5
    redundancy: number;         // 重复度 1-5
    ad_ratio: number;           // 广告占比 1-5
  };
  update_frequency: string;      // 更新频率
  last_seen?: Date;             // 最后检测时间
  last_updated?: Date;          // 最后更新时间
  status: 'normal' | 'inactive' | 'inaccessible' | 'suspicious';
  notes: string;               // 用户备注
  created_at: Date;
  updated_at: Date;
}
```

### 8.2 Rule（规则）

```typescript
interface Rule {
  rule_id: string;
  name: string;
  condition: string;             // 可配置表达式
  action: 'upgrade' | 'downgrade' | 'tag' | 'review';
  action_value?: string;       // 操作值（如新优先级）
  enabled: boolean;
  created_at: Date;
  updated_at: Date;
}
```

### 8.3 Review Log（复核记录）

```typescript
interface ReviewLog {
  log_id: string;
  creator_id: string;
  date: Date;
  user_action: 'upgrade' | 'downgrade' | 'block' | 'note' | 'unfollow';
  reason?: string;
  before_state?: Partial<Creator>;
  after_state?: Partial<Creator>;
}
```

### 8.4 UserSettings（用户设置）

```typescript
interface UserSettings {
  user_id: string;
  auto_unfollow_inactive: boolean;  // 自动取关已注销
  whitelist: string[];              // 白名单ID列表

  notification_enabled: boolean;    // 推送通知
  export_format: 'csv' | 'notion' | 'excel';

  updated_at: Date;
}
```

## 10. V1/V2/V3 增强方向

### V1（3-4周）
- 规则自动分层 + 解释
- AI辅助打标与评分
- 用户反馈学习

### V2（4-6周）
- 半自动导入（OCR/分享链接批量）
- AI视频摘要
- 差异化输出（知识周报/娱乐清单）

### V3（6-8周）
- 跨平台创作者管理（B站/小红书等）
- 知识策展（Timely/Evergreen）
- 优质博主发现
