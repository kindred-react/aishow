## 一、MVP概述

### 1.1 什么是MVP

MVP（Minimum Viable Product，最小可行产品）是指：
- 具有核心功能的最简化版本
- 能够验证产品假设
- 快速交付给用户使用
- 通过用户反馈迭代优化

### 1.2 我们的MVP目标

```
验证核心假设：
1. 用户需要关注治理工具
2. 分层+筛选能解决核心痛点
3. 用户愿意手动管理博主
```

## 四、数据结构（MVP）

### 4.1 博主数据

```typescript
// MVP数据结构
interface Creator {
  creator_id: string;           // 唯一ID
  name: string;                 // 昵称（必填）
  profile_url: string;          // 主页链接（必填）
  platform: 'douyin';            // 平台（固定抖音）
  primary_tag: TagL1;           // L1分类（必填）
  priority: Priority;            // 优先级（必填）
  notes: string;                 // 备注
  status: CreatorStatus;         // 账号状态
  update_frequency: string;       // 更新频率
  last_seen?: Date;            // 最后检测
  created_at: Date;              // 创建时间
  updated_at: Date;              // 更新时间
}
```

### 4.2 枚举定义

```typescript
type TagL1 =
  | 'knowledge'    // 知识/学习
  | 'news'         // 资讯/观点
  | 'skill'        // 技能/教程
  | 'aesthetic'    // 审美/颜值
  | 'entertainment' // 娱乐/抽象
  | 'emotional'     // 情绪价值
  | 'other';        // 其他

type Priority = 'S' | 'A' | 'B' | 'C' | 'D';

type CreatorStatus = 'normal' | 'inactive' | 'unknown';
```

## 五、用户流程

### 5.1 核心用户流程

```
┌─────────────────────────────────────────────────────────────────┐
│                     MVP核心用户流程                              │
└─────────────────────────────────────────────────────────────────┘

1. 导入博主
   用户点击"添加博主" → 填写昵称/链接/备注 → 提交 → 博主入库

2. 打标签
   进入博主详情 → 选择L1分类 → 保存

3. 分层
   进入博主详情 → 选择优先级 → 保存

4. 筛选查看
   进入列表页 → 选择筛选条件 → 查看筛选结果

5. 复盘
   进入复盘中心 → 查看待复核清单 → 执行操作（升/降/确认取关）

6. 导出
   进入列表页 → 点击导出 → 选择范围 → 下载CSV
```

## 八、MVP后的迭代方向

### 8.1 V1方向（基于MVP反馈）

| 功能 | 优先级 | 依据 |
|------|--------|------|
| 账号状态检测 | P1 | 用户反馈异常账号难识别 |
| 规则引擎 | P1 | 用户需要自动化分层 |
| AI辅助评分 | P1 | 用户希望减少手动操作 |

### 8.2 决策依据

```
┌─────────────────────────────────────────────────────────────┐
│                   MVP迭代决策树                            │
└─────────────────────────────────────────────────────────────┘

用户是否愿意手动打标签？
├── 是 ──▶ 用户是否需要自动化分层？
│         ├── 是 ──▶ V1: 规则引擎
│         └── 否 ──▶ 继续优化手动体验
│
└── 否 ──▶ 用户是否接受AI辅助？
          ├── 是 ──▶ V1: AI辅助评分
          └── 否 ──▶ 简化标签体系（减少分类）
```

## 九、风险与预案

| 风险 | 等级 | 预案 |
|------|------|------|
| 用户不愿手动管理 | 🟡 中 | 收集反馈，快速迭代 |
| 分类太复杂 | 🟡 中 | MVP简化到3-4类 |
| 导出格式不符合预期 | 🟡 中 | 支持自定义字段 |
| 列表加载慢 | 🟡 中 | 本地缓存+分页 |
